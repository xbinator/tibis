# 2026-06-19

## Added
- 新增工具元数据 registry，统一记录已迁移工具的名称、runtime、分组、暴露策略和 schema 定义。
- 新增 ChatRuntime 当前架构地图文档，梳理主进程 runtime、renderer bridge、消息生命周期、工具执行边界和后续清理入口。
- 新增 ChatRuntime 主进程 runtime 目录 README，说明核心模块、上下文预算、压缩、工具和常见改动入口。
- 新增 ChatRuntime 主进程工具目录 README，说明工具分组、schema/runtime 边界和新增迁移工具的落点。
- 新增 BChat runtime 事件测试工具，统一 runtime listener 的重置与事件触发，减少 ChatRuntime 测试样板代码。
- ChatRuntime `send` 主进程消息落点：创建并持久化 user 消息与 assistant 占位消息，通过 runtime message 事件同步 renderer。
- ChatRuntime 主进程流式执行器：消费模型 text/reasoning/finish/error chunk，回写 assistant 草稿并在错误时发出 runtime error 事件。
- BChat 普通发送接入主进程 ChatRuntime：renderer 仅发送 runtime 命令，user 与 assistant 消息由主进程创建并通过 runtime 事件同步。
- ChatRuntime 主进程模型上下文转换：从主进程会话消息构建 ModelMessage，保留 user/assistant 历史并从最新成功压缩边界继续上下文。
- ChatRuntime abort 终态：中止主进程模型流、释放会话锁，并将 assistant 草稿标记为“已中断”后同步到 renderer。
- ChatRuntime 上下文用量事件：发送前由主进程基于 ModelMessage 估算上下文 token，并通过 runtime context usage 事件同步 renderer。
- BChat 工具栏上下文用量接入 runtime snapshot：主进程上报后优先显示 runtime 估算，本地估算保留为空闲兜底。
- ChatRuntime 主进程发送前自动压缩：当主进程上下文预算进入 danger 状态时先执行 runtime auto compact，成功后重读压缩边界再启动模型流。
- ChatRuntime 主进程模型上下文支持已完成工具消息：assistant tool-call 与 tool-result 会转换为 AI SDK `ModelMessage`，供后续主进程工具执行迁移复用。
- ChatRuntime 主进程流式执行器支持工具 chunk：tool input/call/result 会写入 assistant tool part 并同步 renderer。
- BChat runtime send 透传主进程可执行上下文配置：memory system prompt、Tavily 配置与 MCP server 快照会随发送命令进入 ChatRuntime。
- ChatRuntime renderer 本地工具桥接：主进程 runtime 可向 BChat 发起工具执行请求，renderer 使用现有工具上下文执行后回传结果，并由主进程继续同一轮模型流。
- ChatRuntime 用户选择续跑：BChat 将用户选择题答案写回消息快照后，通过主进程 `continue` 命令恢复暂停的 assistant 轮次。
- BChat 重新生成接入 ChatRuntime：截断目标 assistant 后通过主进程 `continue` 命令重新生成回复。
- BChat chat 服务配置解析 hook：将模型配置与工具支持能力解析从旧 renderer stream hook 中拆出，供 ChatRuntime 发送与续跑复用。
- ChatRuntime 自动命名命令：主进程负责生成并持久化首轮会话标题，renderer 仅更新当前 UI 镜像与触发列表刷新。

## Changed
- ChatRuntime 工具 registry 源迁移到根目录 `shared/ai/tools/toolRegistry.ts`，renderer 与 Electron main 均直接读取同一份定义源。
- ChatRuntime 工具 registry 改为在 `shared` 内声明跨进程结构类型，避免 renderer 构建解析 `types/ai` 别名失败。
- BChat renderer 侧工具 registry 改用 `.ts` 源文件扩展名，避免 Vite dev server 在浏览器中请求未转译的 `.mts` 源码。
- ChatRuntime 主进程工具分组 Set 集中到 `tools/constants.mts`，各工具目录不再各自维护重复名称集合，并通过测试和工具 registry 保持对齐。
- BChat renderer 内置工具的默认/条件暴露列表改为从工具 registry 的 exposure 策略派生，减少工具名称清单重复维护。
- BChat renderer schema-only 工具工厂新增 registry 派生的 `RUNTIME_TOOL_FACTORIES` 映射，具名工厂统一从映射读取。
- BChat 已迁移工具的 schema-only 工厂改为从工具 registry 派生，`runtimeTools.ts` 不再维护重复 schema 字面量。
- ChatRuntime 主进程新增 `controllers/` 分组目录，将 renderer 工具请求、确认请求和 bridge 请求的 pending 管理从 `service.mts` 拆出。
- ChatRuntime 主进程新增 `messages/` 分组目录，将 runtime 消息工厂和 assistant 终态处理从 `service.mts` 拆出。
- BChat runtime bridge 按 editor、drawing、file、settings、draft/resource 和 dispatcher 加入域分区路标，降低主进程工具 bridge 阅读成本。
- ChatRuntime send 入参增加 `parentRuntimeId`，为后续多 agent 调度链路保留父子 runtime 关系。
- ChatRuntime `send` 改为创建消息后立即返回 runtimeId，模型流在后台执行，使 renderer 可在生成中持有 runtimeId 并发起中止。
- ChatRuntime 模型解析器从摘要专用命名调整为通用 chat model resolver，供流式生成、压缩摘要和自动命名共同复用。
- BChat 本地上下文 token 估算改为字符启发式估算，不再动态加载 tokenizer 包。
- Tavily 主进程工具改为直接调用 Tavily HTTP API，保留 `tavily_search` 与 `tavily_extract` 工具入口。
- AI 服务通用错误映射支持识别 `Insufficient Balance` 等余额不足文案，统一提示为额度耗尽。
- BChat ChatRuntime 续轮命令会在 renderer IPC 边界转换为纯消息快照，避免重新生成时传入 Vue Proxy 导致结构化克隆失败。
- 聊天历史消息排序在同一 `createdAt` 下按角色顺序稳定排序，避免 runtime 同时间戳的 assistant 草稿显示在 user 消息之前。
- BChat ChatRuntime 中断命令改用 void 响应断言，避免主进程成功中断但无返回数据时误报 `ChatRuntime 请求失败`。
- BChat 手动中断不再额外插入独立 interrupt 消息，避免与主进程已更新的 assistant 中断状态重复显示。
- BChat runtime 流式消息同步不再触发滚动到底部，避免阅读历史时被持续拉回底部。
- ChatRuntime 中断终态改为结束原 assistant 草稿并新增独立 `interrupt` 消息，避免同一消息切换角色后视图不刷新。
- BChat 硬中断草稿恢复改为保留原 assistant 消息并新增 `interrupt` 消息，移除未使用的 renderer 侧 interrupt 消息创建器。
- ChatRuntime 上下文压缩取消改为更新既有 compression 消息为 `cancelled` 状态，不再创建聊天中断消息。
- ChatRuntime 中断空 assistant 草稿时改为删除草稿消息并广播删除事件，只保留独立 `interrupt` 消息。
- ChatRuntime assistant 终态 usage 通过 `updateMessage` 写回时同步累计到会话用量，修复 `/usage` 面板无持久化数据的问题。
- BChat `/usage` 打开时会把当前会话 ID 作为刷新兜底，避免侧栏 active session 未同步时跳过加载。
- ChatRuntime 压缩消息事件不再携带 `AbortSignal` 与消息快照等命令字段，修复手动压缩事件无法通过 Electron IPC 同步的问题。
- BChat 手动压缩发送给主进程前会将消息列表转换为可克隆纯快照，并在压缩 IPC 失败时显示错误提示。
- ChatRuntime 模型上下文、续轮 usage 与压缩事件隔离补齐回归修复，避免旧 assistant 内容丢失、多轮工具续跑用量少算以及多 client 压缩消息串线。
- BChat runtime 压缩流程不再弹出 toast，压缩结果统一交给 compression 消息在对话中展示。
- ChatRuntime 遇到 provider 上下文超限时会自动压缩并重放当前用户轮次，重放时将媒体附件降级为文本占位，避免再次触发同类 overflow。
- ChatRuntime 完成后会根据 provider usage 重新计算上下文预算，真实用量超过可用输入预算时自动压缩当前 turn 后的上下文。
- ChatRuntime renderer 本地工具桥接增加失败兜底和超时保护，renderer 不响应或执行失败时写入 tool failure result 并继续模型续轮。
- ChatRuntime 成功 turn 后会剪枝旧的大型 tool output，保留最近用户轮、工具结构和摘要字段，降低长会话上下文膨胀。
- BChat ChatRuntime renderer 工具异常回传会保留稳定错误码，避免确认取消、编辑器不可用等 bridge 错误被统一压成 `EXECUTION_FAILED`。
- BChat ChatRuntime renderer 工具请求在同 client 已切换会话时会立即回传 `EDITOR_UNAVAILABLE`，避免旧会话工具请求被忽略后等到超时。
- BChat 用户选择提交改为走主进程 ChatRuntime `submitUserChoice`，由主进程更新等待中的 tool result 并启动续轮，renderer 不再传整段消息快照。
- ChatRuntime 增加主进程确认桥：main 可发起 `confirmation-requested` 并等待 renderer `submit-confirmation` 决策，BChat 复用现有底部确认弹窗处理 runtime 确认。
- ChatRuntime 增加通用 renderer bridge：main 可发起 `bridge-requested` 并等待 renderer `bridge-response`，为后续 editor/drawing/webview 按需快照和 RPC 工具迁移铺底。
- BChat runtime bridge 支持当前编辑器、画板和 WebView 只读快照，主进程后续可通过 bridge 按需读取 renderer 状态。
- ChatRuntime `read_current_document` 改为主进程工具执行，renderer 仅通过 runtime bridge 提供当前文档快照。
- ChatRuntime `read_current_drawing` 与 `read_current_webpage` 改为主进程工具执行，renderer 仅提供画板和网页快照。
- ChatRuntime `get_current_time` 改为主进程工具执行，避免纯环境只读工具再回调 renderer。
- ChatRuntime `query_logs` 改为主进程工具执行，直接读取主进程日志文件并保留原查询筛选语义。
- ChatRuntime `read_file` 与 `read_directory` 开始迁移到主进程执行，支持工作区内文件系统读取，并通过 renderer bridge 优先读取已打开编辑器最新内容。
- ChatRuntime `read_file` 支持通过 renderer bridge 读取 `unsaved://` 未保存草稿内容，文件和目录读取工作区外路径时复用主进程确认桥。
- ChatRuntime `get_settings` 与 `open_resource` 改为主进程工具执行，renderer 仅通过 bridge 提供设置快照和资源打开动作。
- ChatRuntime `update_settings` 改为主进程工具执行，主进程负责设置写入确认，renderer bridge 仅应用设置修改。
- ChatRuntime `create_document` 改为主进程工具执行，renderer bridge 仅负责创建并打开未保存草稿。
- ChatRuntime `write_file` 改为主进程工具执行，主进程负责路径解析、确认和磁盘写入，renderer bridge 仅处理已打开编辑器和未保存草稿内容写入。
- ChatRuntime `edit_file` 改为主进程工具执行，主进程负责精确匹配、替换和确认，复用文件内容 bridge 写回编辑器或草稿。
- 新增并细化 ChatRuntime 主进程工具迁移 TODO 文档，记录已完成工具、剩余任务、代码落点和验证命令。
- Drawing AI 工具抽出 `drawingRuntime` 纯逻辑模块，供 renderer 工具和主进程 ChatRuntime 复用。
- ChatRuntime `create_drawing` 改为主进程工具执行，主进程复用 Drawing 纯逻辑生成 `.tibis` 草稿内容，renderer bridge 仅创建未保存草稿。
- ChatRuntime `apply_drawing_operations` 改为主进程工具执行，主进程读取画板快照、应用 Drawing operations、确认后通过 renderer bridge 写回当前画板。
- ChatRuntime `get_mcp_settings` 改为主进程工具执行，主进程直接读取 Tibis settings 文件并返回 MCP server 配置快照。
- ChatRuntime MCP 写工具改为主进程执行，`add_mcp_server`、`update_mcp_server`、`remove_mcp_server` 直接确认后更新 settings 文件，`refresh_mcp_discovery` 直接调用主进程 MCP session 刷新工具发现。
- ChatRuntime 主进程新增独立 Drawing runtime 纯逻辑与边界测试，避免 Electron main 编译直接引用 renderer `src` 模块并在源码目录生成 `.js` 产物。
- ChatRuntime 主进程工具按 `tools/**/index.mts` 文件夹结构拆分，`service.mts` 仅保留 runtime 生命周期、bridge/confirmation 管理和 main tool executor 注入。
- BChat runtime bridge 的未保存文件读取依赖支持异步最近文件查询，修复 `filesStore.getFileById` 接入时的类型不匹配。
- BChat 内置工具注册改为通过 `src/ai/tools/catalog/runtimeTools.ts` 暴露已迁移工具的 schema-only 定义，renderer `builtin` 目录不再保留这些工具的本地执行逻辑。
- ChatRuntime 主进程文件读取和资源打开会拒绝逃逸工作区的相对路径，避免 `../` 绕过工作区外确认与读取边界。
- ChatRuntime 主进程 renderer 确认、bridge 与本地工具请求增加超时清理，避免 renderer 无响应时 runtime 长时间悬挂。
- BChat runtime 消息事件写入本地列表时会按会话展示顺序排序，避免跨 IPC 事件先后不稳定导致 user/assistant 显示错位。
- BChat 重新生成使用的 assistant 占位消息不再写入空创建时间，ChatRuntime 续跑时也会兜底补齐旧空时间，避免 assistant 被排序到用户消息之前。
- ChatRuntime 工具结果为取消状态时不再进入下一轮工具续跑，避免用户尚未确认或取消某个工具后继续执行后续命令。
- ChatRuntime 主进程确认请求不再按技术超时自动视为取消，避免用户尚未处理确认弹窗时工具被误取消。
- BChat 确认控制器改为串行队列，避免新确认请求覆盖并自动取消尚未处理的当前确认。
- BChat 普通发送不再在 renderer 本地追加 user 消息，统一等待主进程 `message-created` 事件同步，避免消息 owner 分裂造成重复或错序。
- BChat 重新生成不再在 renderer 本地创建或持久化 assistant 占位，ChatRuntime `continue` 在缺少 assistant 快照时由主进程创建占位消息。
- BChat 上下文压缩 hook 移除 toast 依赖，压缩失败、取消和跳过状态统一由 compression 消息或静默返回表达。

## Removed
- 移除已无生产入口的 BChat 旧本地压缩 hook `useCompactContext` 及其旧集成测试，当前 `/compact` 仅保留主进程 ChatRuntime 压缩入口。
- 移除 BChat renderer 发送前基于本地 `usedTokens` 的自动压缩触发，避免与主进程 ChatRuntime 自动压缩重复执行。
- 移除 `useChatStream` 中已迁移到 ChatRuntime 的 `submitUserChoice` 与 `regenerate` 旧入口。
- 移除 BChat renderer 本地 `useChatStream` 流式 hook 及其旧草稿持久化测试，BChat 生成流程改为仅依赖主进程 ChatRuntime。
- 移除 `js-tiktoken` 与 `@tavily/ai-sdk` 依赖，避免安装树继续引入 tokenizer 包。
- 移除已迁移到 ChatRuntime 主进程的 renderer 内置工具执行器目录，包括 Document、Drawing、Environment、File、Logs、MCP、OpenResource、Settings 与 Webpage。
