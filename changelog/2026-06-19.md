# 2026-06-19

## Added
- ChatRuntime `send` 主进程消息落点：创建并持久化 user 消息与 assistant 占位消息，通过 runtime message 事件同步 renderer。
- ChatRuntime 主进程流式执行器：消费模型 text/reasoning/finish/error chunk，回写 assistant 草稿并在错误时发出 runtime error 事件。
- BChat 普通发送接入主进程 ChatRuntime：renderer 仅创建本地 user 展示并发送 runtime 命令，后续消息由 runtime 事件同步。
- ChatRuntime 主进程模型上下文转换：从主进程会话消息构建 ModelMessage，保留 user/assistant 历史并从最新成功压缩边界继续上下文。
- ChatRuntime abort 终态：中止主进程模型流、释放会话锁，并将 assistant 草稿标记为“已中断”后同步到 renderer。
- ChatRuntime 上下文用量事件：发送前由主进程基于 ModelMessage 估算上下文 token，并通过 runtime context usage 事件同步 renderer。
- BChat 工具栏上下文用量接入 runtime snapshot：主进程上报后优先显示 runtime 估算，本地估算保留为空闲兜底。
- ChatRuntime 主进程发送前自动压缩：当主进程上下文预算进入 danger 状态时先执行 runtime auto compact，成功后重读压缩边界再启动模型流。
- ChatRuntime 主进程模型上下文支持已完成工具消息：assistant tool-call 与 tool-result 会转换为 AI SDK `ModelMessage`，供后续主进程工具执行迁移复用。
- ChatRuntime 主进程流式执行器支持工具 chunk：tool input/call/result 会写入 assistant tool part 并同步 renderer。
- BChat runtime send 透传主进程可执行上下文配置：memory system prompt、Tavily 配置与 MCP server 快照会随发送命令进入 ChatRuntime。
- ChatRuntime renderer 本地工具桥接：主进程 runtime 可向 BChat 发起工具执行请求，renderer 使用现有工具上下文执行后回传结果，并由主进程继续同一轮模型流。
- ChatRuntime 用户选择续跑：BChat 将用户选择题答案写回消息快照后，通过主进程 `continue` 命令恢复暂停的 assistant 轮次。
- BChat 重新生成接入 ChatRuntime：截断目标 assistant 后创建新占位消息，并通过主进程 `continue` 命令重新生成回复。
- BChat chat 服务配置解析 hook：将模型配置与工具支持能力解析从旧 renderer stream hook 中拆出，供 ChatRuntime 发送与续跑复用。
- ChatRuntime 自动命名命令：主进程负责生成并持久化首轮会话标题，renderer 仅更新当前 UI 镜像与触发列表刷新。

## Changed
- ChatRuntime send 入参增加 `parentRuntimeId`，为后续多 agent 调度链路保留父子 runtime 关系。
- ChatRuntime `send` 改为创建消息后立即返回 runtimeId，模型流在后台执行，使 renderer 可在生成中持有 runtimeId 并发起中止。
- ChatRuntime 模型解析器从摘要专用命名调整为通用 chat model resolver，供流式生成、压缩摘要和自动命名共同复用。
- BChat 本地上下文 token 估算改为字符启发式估算，不再动态加载 tokenizer 包。
- Tavily 主进程工具改为直接调用 Tavily HTTP API，保留 `tavily_search` 与 `tavily_extract` 工具入口。
- AI 服务通用错误映射支持识别 `Insufficient Balance` 等余额不足文案，统一提示为额度耗尽。
- BChat ChatRuntime 续轮命令会在 renderer IPC 边界转换为纯消息快照，避免重新生成时传入 Vue Proxy 导致结构化克隆失败。
- 聊天历史消息排序在同一 `createdAt` 下按角色顺序稳定排序，避免 runtime 同时间戳的 assistant 草稿显示在 user 消息之前。
- BChat ChatRuntime 中断命令改用 void 响应断言，避免主进程成功中断但无返回数据时误报 `ChatRuntime 请求失败`。
- BChat 手动中断不再额外插入独立 interrupt 消息，避免与主进程已更新的 assistant 中断状态重复显示。
- BChat runtime 流式消息同步不再触发滚动到底部，避免阅读历史时被持续拉回底部。
- ChatRuntime 中断终态改为结束原 assistant 草稿并新增独立 `interrupt` 消息，避免同一消息切换角色后视图不刷新。
- BChat 硬中断草稿恢复改为保留原 assistant 消息并新增 `interrupt` 消息，移除未使用的 renderer 侧 interrupt 消息创建器。
- ChatRuntime 上下文压缩取消改为更新既有 compression 消息为 `cancelled` 状态，不再创建聊天中断消息。
- ChatRuntime 中断空 assistant 草稿时改为删除草稿消息并广播删除事件，只保留独立 `interrupt` 消息。
- ChatRuntime assistant 终态 usage 通过 `updateMessage` 写回时同步累计到会话用量，修复 `/usage` 面板无持久化数据的问题。
- BChat `/usage` 打开时会把当前会话 ID 作为刷新兜底，避免侧栏 active session 未同步时跳过加载。
- ChatRuntime 压缩消息事件不再携带 `AbortSignal` 与消息快照等命令字段，修复手动压缩事件无法通过 Electron IPC 同步的问题。
- BChat 手动压缩发送给主进程前会将消息列表转换为可克隆纯快照，并在压缩 IPC 失败时显示错误提示。
- ChatRuntime 模型上下文、续轮 usage 与压缩事件隔离补齐回归修复，避免旧 assistant 内容丢失、多轮工具续跑用量少算以及多 client 压缩消息串线。
- BChat runtime 压缩流程不再弹出 toast，压缩结果统一交给 compression 消息在对话中展示。

## Removed
- 移除 BChat renderer 发送前基于本地 `usedTokens` 的自动压缩触发，避免与主进程 ChatRuntime 自动压缩重复执行。
- 移除 `useChatStream` 中已迁移到 ChatRuntime 的 `submitUserChoice` 与 `regenerate` 旧入口。
- 移除 BChat renderer 本地 `useChatStream` 流式 hook 及其旧草稿持久化测试，BChat 生成流程改为仅依赖主进程 ChatRuntime。
- 移除 `js-tiktoken` 与 `@tavily/ai-sdk` 依赖，避免安装树继续引入 tokenizer 包。
