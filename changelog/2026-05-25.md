# 2026-05-25

## Added
- 新增 Tibis 工作区根目录提供器（`electron/main/modules/workspace/root.mts`），默认路径 `~/.tibis`，自动创建并 realpath 规范化
- 新增 IPC handler `workspace:get-root`，供渲染进程获取工作区根目录
- Native Bridge 新增 `getTibisWorkspaceRoot()` 方法，Electron 端调用 IPC，Web 端返回 null
- Shell 工具新增 Tibis 工作区回退逻辑：调用方未提供 workspaceRoot 时自动使用 `~/.tibis`
- 新增工作区根目录测试（4 个用例）和 Shell 工具回退测试（2 个用例）
- MCP SDK 迁移：安装 `@modelcontextprotocol/sdk`，新增 StreamableHTTP/SSE transport 支持
- 新增 MCP OAuth 2.0 认证流程（Provider、回调服务器、Token 持久化存储）
- 新增 MCP 工具变更通知监听（`notifications/tools/list_changed`）
- 新增 MCP 会话管理模块（session.mts），替代原 runtime.mts
- 新增 MCP 状态管理（status.mts），支持 `needs_auth`/`needs_client_registration` 状态
- 新增 MCP 错误分类（errors.mts），自动映射 SDK 错误到应用状态
- 新增 MCP discovery 缓存模块（discovery.mts）
- 新增 IPC 通道 `tools:mcp:oauth:start`、`tools:mcp:oauth:clear`、`tools:mcp:tools-changed`、`tools:mcp:oauth:open-url`
- MCP 设置页支持 transport 类型选择（stdio/streamableHTTP/sse）、URL 配置、OAuth 开关
- MCP 设置页下拉菜单新增 OAuth 认证和清除凭据操作

## Changed
- Shell 安全检查放行 `/dev/null`（bash）和 `$null`/`Out-Null`（PowerShell）重定向目标，不再误判为工作区外写入
- Shell 工具描述和参数说明从"当前工作区/工作区根目录"改为"Tibis 工作区/Tibis 工作区目录"
- Shell 工具缺少 workspaceRoot 时的错误消息改为"无法初始化 Tibis 工作区目录"
- 确认描述文案改为"AI 请求在 Tibis 工作区内执行 Shell 命令"
- MCP 类型定义更新：`MCPServerConfig` 新增 `transport`、`url`、`oauth`、`watchToolChanges` 字段
- MCP 设置页 server 列表显示 transport 类型和 URL（远程模式）
- AI 服务 MCP 判断逻辑更新：支持远程 transport 的可用性检查
- `electron-api.d.ts` 新增 `startMcpOAuth`、`clearMcpOAuth`、`onMcpToolsChanged`、`onMcpOAuthOpenUrl` 接口
- MCP 测试文件更新：适配新 session/discovery/errors/status 模块，新增 OAuth IPC 通道断言
- 修复 OAuth 模块导入路径（`storage.mts`、`callback-server.mts` 中 logger/store 相对路径修正）
- 修复 `notifications.mts` 使用 SDK 导出的 Zod Schema 注册通知处理器
- 修复 `provider.mts` 从 `shared/auth.js` 导入 `OAuthClientInformationMixed`/`OAuthTokens`
- 修复 `session.mts` 缺失 `clearAllDiscoveryCache` 导入

## Removed
- 删除旧 MCP runtime 模块（`runtime.mts`、`local-stdio.mts`），由 session.mts 替代
