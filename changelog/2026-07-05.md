# 2026-07-05

## Added
- BSection Item 新增 `tips` prop，仅启用 ATooltip 包裹（不加虚线下划线视觉提示）；与 `tooltip` 区别在于无 `--tooltip` 修饰符，二者同时传入时 ATooltip 文本优先使用 `tooltip`。
- BSection Item 新增 `labelAlign` prop（`'left' | 'center' | 'right'`，默认 `left`），支持标签在 `labelMinWidth` 剩余空间内水平居中 / 右对齐。
- BSection Item 新增 `contentAlign` prop（`'left' | 'right'`，默认 `left`），支持默认插槽内容（控件区）贴右排列：horizontal 模式下用 `margin-left:auto`，vertical 模式下用 `align-self:flex-end`。
- 新增 BInputNumber 数字输入组件，封装 AInputNumber 并默认隐藏增减按钮（controls=false）。
- BInputNumber 新增 `defaultValue`（空值兜底）和 `decimalPrecision`（输出小数精度）归一化 props，内置值格式化逻辑。
- BInputNumber 输入时允许自由输入，`update:value` 时实时归一化向外 emit。
- 用 `isInternalUpdate` 标记区分内部 / 外部 modelValue 变化，内部触发时跳过 watch 同步，避免归一化值覆盖用户输入的中间状态。
- 提取 `normalizeOutputValue` 为独立工具函数（`utils/normalize.ts`），便于测试。
- 新增 BInputNumber 归一化函数单元测试（23 cases）和组件集成测试（11 cases）。
- Widget 元素样式新增 `lineHeight`（行高）、`textDecoration`（文字修饰线）和 `fontStyle`（斜体）字段，新增 `WidgetTextDecoration`、`WidgetFontStyle` 类型。
- DesignSetter 文字区块新增行高输入、斜体选择和文字修饰分段控件（无 / 下划线 / 删除线）。
- Widget 元素新增持久化 `locked` 字段，用于锁定元素位置和尺寸。
- DesignSetter 布局区新增锁定按钮，锁定后禁用 X/Y/宽/高输入。
- BWidget 运行态沙箱新增 `$logger` 宿主通道：用户脚本可调用 `this.$logger.info/warn/error` 把日志写入应用日志文件，在「设置 → 日志」页面查看，message 自动带 `[widget]` 前缀以区分来源。
- Widget 脚本编辑器（CodeEditor）Monaco 类型提示新增 `console`（log/info/warn/error/debug）和 `$logger` 接口声明，解决 `lib: ['es2020']` 不含 DOM lib 导致无补全的问题。
- 新增 `widgetRuntime/logger.ts` 工具，把 `unknown[]` 序列化为单行字符串（Error 取 name+message，对象走 JSON.stringify 并对循环引用 fallback）。
- 新增 `widget-runtime-logger.test.ts`，覆盖 `formatWidgetLogArgs` 纯函数（10 cases）与 `$logger` 沙箱桥接（10 cases：info/warn/error 透传、data patch 先 flush、无参数、深拷贝断引用、Error/undefined 保真、循环引用/BigInt 不触发 failure、unmounted 钩子、脚本抛错/禁用不触发）。
- BWidget 运行态沙箱新增 `console` 桥接通道：用户脚本内 `console.log/info/warn/error/debug` 通过 `__sandboxWidgetConsole` 宿主函数转发到主线程，由 `Runtime.vue` 的 `handleWidgetConsole` 输出到 DevTools；沙箱执行栈销毁后主线程仍可在控制台展开对象内容。
- `widget-runtime-logger.test.ts` 补充 `console` 桥接测试（7 cases：log 透传、info/warn/error/debug 全级别、对象 args 深拷贝断引用、undefined/BigInt 不触发 failure、`$logger` 与 `console` 独立、unmounted 钩子、脚本禁用不触发）。
- `PanelSidebar` 侧边栏新增「数据源」与「动作」两个 tab 占位（`lucide:database` / `lucide:zap`）。
- 新建 `SidebarDataSource.vue` 与 `SidebarAction.vue` 占位组件：与 `SidebarLayer` 风格一致的图标 + 提示文案空态，零数据契约变更，后续承载真实功能时再替换。

## Changed
- `$logger` 相关命名对齐：`WidgetLogger` 接口改为 `Logger`；`logFormat.ts` 重命名为 `logger.ts`；`onLog` 选项改为 `onLogger`；`handleWidgetLog` 改为 `handleWidgetLogger`；`SANDBOX_WIDGET_LOG_HOST_FUNCTION_NAME` 改为 `SANDBOX_WIDGET_LOGGER_HOST_FUNCTION_NAME`（沙箱胶水代码内 `__sandboxWidgetLog` 同步改为 `__sandboxWidgetLogger`）。
- `createWidgetElementContentStyleProperties` 输出 `lineHeight`、`textDecoration` 和 `fontStyle` CSS 属性。
- `measureWidgetTextElementSize` 文本测量在用户设置 `lineHeight` 时使用自定义行高，否则回退到默认比例，保证测量与渲染一致。
- 将 DesignSetter、ControlPanel、AdvancedSetter、BatchSetter、Text/Setter 中的 AInputNumber 替换为 BInputNumber，移除冗余的 `:controls="false"`。
- ControlPanel 的 ValueType 类型改为从 BInputNumber/types.ts 导入，替代 ant-design-vue 内部路径。
- 移除 Text/Setter 中 AInputNumber 的无效 `allow-clear` 属性。
- DesignSetter 布局字段改用 BInputNumber 的 `defaultValue` + `decimalPrecision` 替代手动 `normalizeLayoutInputValue`，移除 `NumericInputValue` 类型和归一化函数。
- BWidget 画布移动、直接拖拽和缩放入口会忽略锁定元素的几何变更。
- Widget 锁定仅限制被锁元素作为直接操作目标；父组合整体移动或缩放时，锁定子元素会随组合一起变换。
- `__widgetReservedBindingNames` 新增 `console` 和 `widgetThis`，避免用户 methods 中同名方法与交互脚本注入参数冲突导致「沙箱变量名重复」错误。
- 重命名 `SidebarDataSource.vue` 为 `SidebarState.vue`：组件名、文件头注释、模板 CSS 类名（`data-source-empty*` → `state-empty*`）以及 `PanelSidebar.vue` 中的引用同步更新。

## Fixed
- 修复 `runPartSandbox` 未把 `onLogger` 透传给 `runWidgetScriptSandbox`，导致 `$logger.info/warn/error` 调用无法触达宿主回调的问题（被新增测试发现）。
- 修复 `$logger` 与 `console` 沙箱桥接复用 JSON 克隆导致 `Error` 变为空对象、`undefined` 变 `null`，以及循环引用或 `BigInt` 参数触发运行态 failure 的问题。
- 修复 Widget 设计面板清空 X/Y/宽/高布局输入时写入空值，导致节点在画布中消失的问题。
- 修复多选布局面板可绕过锁定状态修改直接选中元素位置和尺寸的问题。
- 修复小组件脚本 `console.log` 输出的对象在 DevTools 无法展开查看的问题：沙箱执行栈销毁后 Worker 上下文对象引用失效，通过 `__sandboxWidgetConsole` 宿主函数把 `console` 调用桥接到主线程，args 直接交给通用沙箱宿主桥接做结构化克隆后在主线程 DevTools 输出。

## Removed
- 移除 `PanelSidebar` 中 `handleSplitterClose` 钩子及对应 `BPanelSplitter` 的 `@close` 监听：splitter 关闭后保留当前 tab 选中态，仅靠内容区宽度为 0 隐藏面板；点击 tab 时由 `handleTabClick` 恢复默认宽度。同步删除 `panel-sidebar.test.ts` 中 `splitter 关闭清空 active tab` 测试用例。
