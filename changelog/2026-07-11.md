# 2026-07-11

## Changed

- BWidget 元素展示字段仅在 Runtime 模式解析变量模板，设计态隐藏 `{{ }}` 变量占位。
- BWidget 渲染上下文扩展参数统一收敛到 `provideRenderContext` 的 options 对象。
- BWidget 渲染上下文读取移除专用 options/mode helper，统一通过 `useRenderContext` 返回对象读取。
- 沙箱 Worker 宿主函数桥接改为按当前活跃运行 ID 路由，修复 Widget session 复用后交互方法执行超时。
- 沙箱 Worker 入口、宿主桥接与协议运行时统一迁移至 `src/utils/sandbox/worker` 目录。
- 沙箱 Session 补齐 Worker 运行串行、pending 宿主调用清理和 dispose 后队列拦截，避免跨运行串线与销毁后继续执行。
- Widget 运行态 Session 每轮同步最新输入输出和数据，固定 Widget 原型边界，并等待托管异步任务及上报未处理的 helper 异常。
- Widget Runtime 卸载后使排队任务失效，避免旧任务重新创建 Session 或继续提交运行态变化。
- Skill、Widget 与文档工具统一采用事实源实时读取，修复源文件更新后聊天继续使用旧数据的问题。
- BChat 聊天流程迁移到 XState Supervisor、Session、Turn、Primary Agent actor 树，支持会话切换后后台任务继续运行，并明确当前不承载未实现的子 Agent 动态编排。
- BChat 输入、会话生命周期、Runtime Bridge 与工作流编排下沉到 hooks 和 `src/ai/chat`，组件仅保留页面装配与 UI 事件绑定。
- Runtime capability 按 runtime ID 冻结，后台确认交互可在切回会话后恢复，发送、续轮、取消、压缩和回退统一同步到 Session 状态机。
- ChatRuntime 新增活跃任务恢复快照与 IPC，renderer 重建后可恢复 actor 路由、待确认交互和 capability 身份，并对无法恢复的本地工具请求返回明确失败。
- BChat Runtime 事件统一由应用级 listener 消费，移除组件任务锁；组件卸载与会话切换不再中止主进程后台任务。
- ChatRuntime 改为由 renderer 预分配 Runtime ID 并在 IPC 前注册 Actor 路由，修复启动事件丢失、文件 Bridge 超时和启动中取消产生孤儿任务的问题。
- Runtime 确认授权统一由应用级 listener 与共享策略处理，恢复会话级和永久授权的自动批准；删除组件级重复 listener、不可达恢复状态和 XState 空转状态。
- BChat Runtime 请求 hook 接受服务配置缺失时的 `undefined` 返回值，并按只读可空 Ref 读取工作区路径，修复 Vue 类型检查错误。
- TodoPanel 取消 LLM 写入待办时自动展开面板的行为，改为默认收起仅显示摘要条，由用户手动点击展开；移除 `useTodoPanel` 中 `shouldAutoShowTodoPanel` 自动展开分支。
- Skill 创建流程过滤安装临时/隐藏目录事件，校验导入名称与资源路径的 Windows 兼容性，并避免重复点击与刷新失败造成误判安装失败。
- Widget 创建流程补充 Windows 保留标识校验、隐藏目录监听过滤和创建防重入，避免异常路径或连续点击导致写入异常。
