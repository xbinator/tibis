# 2026-06-22

## Added
- 新增 ChatRuntime WebView Agent 操作能力，支持通过 `operate_webpage` 对当前激活 WebView 执行点击、输入、选择、滚动、页面导航和等待操作。
- 新增 WebView Agent 控制设计文档与实施计划，明确不引入 MCP、不暴露第三方插件、仅在 WebView 激活时开放工具。

## Changed
- `Modal.input` 新增 `autofocus` 配置项（默认 `true`），底层改为通过 `v-focus` 指令实现自动聚焦，调用方可在弹窗打开时控制是否聚焦输入框。
- 优化 BChat 消息列表 `v-memo` 依赖生成逻辑，收敛模板表达式并补充禁用态、回退状态等子组件展示依赖。
- `read_current_webpage` 升级为网页观察快照，返回 `snapshotId`、加载状态、滚动信息和可交互元素列表，供 `operate_webpage` 使用。
- `operate_webpage` 确认卡支持“本会话允许”和“始终允许”，并复用现有工具授权记录自动批准后续同类 WebView 操作。
- `operate_webpage` 新增 `press` 动作，支持在网页输入框等可编辑元素上模拟 Enter、Tab、Esc 和方向键。
- 基础设置页新增 AI 工具权限管理入口，支持查看始终允许工具、撤销单项授权和清除全部始终允许授权。
- Chat 运行时仅在当前存在激活 WebView 上下文时暴露 `read_current_webpage` 与 `operate_webpage`，并隐藏 `open_resource`。
- `read_current_webpage` 新增页面视口提示、简化 DOM 结构和底部滚动提示，让模型更容易理解网页结构与可操作目标上下文。
- `read_current_webpage` 支持识别带点击意图线索的非语义元素，并在简化 DOM 中保留 `class` 与常见 `data-*` 定位属性。
- `read_current_webpage` 新增当前视口视觉摘要，可标注顶层弹窗/浮层、遮罩背景和主操作按钮，帮助模型优先点击当前可见对话框内元素。
- `read_current_webpage` 与 `operate_webpage` 支持遍历 open Shadow DOM，使 Web Component 内的可见按钮可被读取并通过 index 操作。
- `read_current_webpage` 返回用户手动选中的页面元素摘要，并在可匹配时标注对应可操作元素 index。
- WebView 地址栏将“打开开发者工具”迁移到更多操作菜单，并在存在已选中页面元素时提供支持滚动拼接的“选中元素”完整截图入口。
- WebView DOM 检查面板（InspectorPanel）头部新增关闭按钮，触发 `close` 事件以通知父组件隐藏面板。
- 优化最近记录搜索弹窗结果项样式，改为单行紧凑透明列表，并为文件、路径候选项补充类型图标；网页记录在搜索弹窗和欢迎页中统一使用 GeoJSON 图标，`.json` / `.tibis` 文件使用标准 JSON 图标，`package.json` 显示 npm 图标，欢迎页最近记录同步复用统一文件图标逻辑。

## Removed
- 移除 WebView 页面元素选中框内的截图快捷按钮，保留更多操作菜单中的选中元素截图入口。

## Fixed
- 修复 ChatRuntime 工具续轮期间提前将 assistant 消息标记为完成，导致消息仍在生成时显示完成态工具栏的问题。
- 修复模型未返回内容即中止后，用户消息后仅有中断提示时不显示回退按钮的问题。
- 修复等待用户补充提问工具输入时主动中止未显示中断状态的问题。
- 修复中止后重新生成当前回复时误复用上一轮 assistant，导致 loading 显示到上一条模型消息的问题。
- 修复 AI SDK 6 工具输入事件使用 `toolCallId` / `inputTextDelta` 字段时未被识别，导致提问工具输入流和执行状态异常的问题。
- 修复等待用户选择时 assistant 保持 `finished: false` 后，消息列表 `v-memo` 未感知工具状态变化导致提问工具界面停留在转圈状态的问题。
- 修复 QuestionCard 点击取消被聊天任务锁拦截的问题，并将取消选择作为工具结果继续交给大模型回复。
- 修复提示词编辑器文件提及与 Slash 命令菜单键盘上下切换时活动项未自动滚入可视区的问题，并保持鼠标悬停只更新高亮不触发滚动。
- 修复 Rich 编辑器表格单元格内 AI 选区高亮切分相邻 CJK 文本后触发自动换行的问题。
- 修复编辑器选区“插入对话”在聊天侧栏尚未挂载或未保存文件引用时无法插入到聊天输入框的问题。
- 修复 BEditor 内联补全旧请求晚返回时误取消新补全结果的问题。
- 修复 WebView 选中元素滚出可视区域后无法读取截图位置、超大元素超过页面可截图边界时报错，并避免选中框出现在截图中的问题。
- 修复 WebView 选中元素位于页面内部滚动容器时截图仍错误滚动 body，导致只能截取当前可视区域的问题。
- 修复 WebView 选中元素截图被无关 fixed/sticky 顶部栏遮挡，以及选中 fixed/sticky 元素本身时被滚动裁错的问题。
- 修复 WebView DOM 元素选择器不唯一时，选中列表后续项截图却重新定位到第一个同类元素的问题。
- 修复 WebView 选中元素截图在重复 `id`、未吸附 sticky、视口外 fixed/sticky 与异常清理路径下的边界问题。
- 修复 WebView 同父级重复 `id` 元素生成的选择器仍可能命中第一个元素，并限制协议截图只能截取当前宿主 WebContents 持有的 WebView。
- 修复 WebView Agent 操作的快照过期、页面导航竞态、select 同名选项歧义、scroll 不可滚动目标和 bridge 超时错误码处理。
- 修复 AI 打开当前 WebView 页面时倾向调用 `open_resource` 的问题，改为通过 `operate_webpage` 的 `navigate` 动作在当前 WebView 内导航。
- 修复 `operate_webpage` 滚动动作选择错误滚动容器导致页面未滚动的问题，改为按滚动方向选择可移动祖先并在必要时回退到页面滚动。
- 修复 `operate_webpage` 基于旧快照 index 误操作新 DOM 元素的问题，并让滚动动作返回实际滚动目标与前后位置。
- 修复 `operate_webpage.navigate` 仍依赖页面快照的问题，并收窄 WebView 快照元素动作列表以减少无效操作尝试。
- 修复 WebView 导航被新导航中断时 `loadURL` 的 `ERR_ABORTED` Promise rejection 变成未捕获异常的问题。
- 修复 WebView 地址栏“清除缓存”仅清 HTTP 缓存的问题，改为同步清理 Cookie、localStorage、sessionStorage 与持久化站点数据。
- 补充 `operate_webpage` 在聊天工具调用与结果摘要中的中文展示文案。
- 优化 WebView 截图体验，完整可见的选中元素改为原生裁剪避免滚动，跨视口/完整页面拼接时显示宿主层遮罩降低页面跳动感。
- 优化 WebView 选中元素截图体验，普通页面元素优先通过 Electron 协议层截取视口外区域，减少滚动拼接造成的跳动和闪动。
- 优化 WebView 截图交互，截图中禁用截图菜单并延迟显示宿主层遮罩，短截图不再闪烁遮罩。
- 重构 WebView 截图逻辑，将宿主层截图遮罩默认管理收敛到 `useScreenshot`，减少页面入口组件中的截图细节。
