# 2026-06-23

## Changed
- WebView 最近记录写入页面 favicon，并在欢迎页与最近记录搜索弹窗中优先展示站点图标，缺失或加载失败时回退到默认网页图标。
- 抽离最近记录图标为 `BRecentIcon` 组件，统一处理文件图标、WebView favicon 与默认回退图标；`addWebviewRecord` 的 favicon 入参改为 options 对象。
- 将最近记录搜索弹窗组件从 `BSearchRecent` 重命名为 `BRecent`，并将图标组件迁移到 `src/components/BRecent/Icon.vue`。
- 文件提及菜单改用 `BRecentIcon` 渲染文件图标，确保 `package.json`、`.tibis` 等特殊文件名与最近记录列表使用同一套图标逻辑。
- HeaderTabs 标签页接入 `BRecentIcon`，文件标签显示文件类型图标，WebView 标签优先显示最近记录 favicon。
- 聊天用户消息和输入框文件引用 chip 接入 `BRecentIcon`，让引用文件与最近记录、标签栏使用统一图标。
- 聊天文件引用 token 统一为 `{{@path}}`、`{{@path#L644}}`、`{{@path#L644-L685}}`，程序生成不再输出 `[](...)` 编码路径片段。
- 修复 `{{@path}}` 文件引用 token 误触发输入框 @ 文件提及菜单的问题，并统一消息解析与输入框 chip 对跨行 token 的识别规则。
- 聊天运行时缺失文件错误统一改为中文提示，并复用到 IPC 返回、异步错误事件与上下文压缩错误链路，避免将 `ENOENT` 系统错误直接展示给用户。
- 聊天发送失败与运行时错误改为写入当前会话错误消息，避免普通消息失败只显示临时 toast。
- 聊天启动前错误消息改为独立错误角色并隐藏重新生成入口，模型生成中的 assistant 错误仍保留重新生成功能。
- 移除 BChat 内部 `fileReferenceContext`，将文件引用匹配和内容抽取统一迁移到 `src/utils/file/reference.ts`。
- 聊天输入框 @ 文件提及列表仅展示 Markdown 最近文件，避免插入暂不支持的文件类型。
- 聊天对话滚动到底部按钮在远离底部且模型生成中保持可见，滚到底部时仍按滚动状态隐藏。
- BPanelSplitter 拖拽期间增加全屏透明拖拽屏蔽层，避免鼠标在 WebView 上松开后拖拽状态无法结束。
- 修复 BPanelSplitter 拖拽屏蔽层导致外部 class 无法透传的问题，恢复 ChatSider 放大态样式。
- 统一聊天上下文用量估算与预算口径到 `shared/ai/context`，前端展示与主进程发送前判断复用同一套 token 权重、输出预留和阈值。
- 将 Rich 表格内联 AI 选区的 CSS Custom Highlight 样式改为运行时注入，避免 Lightning CSS 构建警告。
- 增强 WebView Agent 可操作元素识别，补充 BrowserState 风格 `summary`、可点击置信分、推断原因、语义路径、中心点命中目标，并让点击优先派发到命中的子节点。
- 收紧 WebView Agent `navigate` 契约，仅用于用户明确给出 URL 的地址栏导航，页面可见项必须使用 `read_current_webpage` 的 `[N]` 句柄操作。
- 修复 WebView 页面短暂 loading 事件导致刚读取的 Agent 快照立即过期的问题，并避免小型固定工具条被误判为顶层浮层。
- 增强 WebView Agent 顶部短文本入口识别，支持无语义容器、无显式点击样式的紧凑工具栏文本被标记为可操作元素。
- 优化 WebView Agent `summary/content` 简化 DOM 序列化，避免父容器合并子元素文本形成误导性的单一可点击描述。
- 将 WebView 自动化的快照脚本、操作脚本、规范化逻辑和内部类型拆分到 `src/views/webview/web/automation`，降低 `useWebView.ts` 职责复杂度。
- 重构 WebView `read_current_webpage` 页面读取链路，改为共享扁平 DOM 树采集、page-agent 风格简化 DOM 输出，并让 `operate_webpage` 复用同一套元素索引规则。
- 增强 WebView DOM 引擎交互判定，补充 page-agent 风格 cursor、ARIA role、事件属性、链接子元素、多点命中测试与嵌套附属元素过滤。
- 为 WebView Agent 读取与操作过程增加主题色呼吸边框和状态提示，用户可感知当前网页正在被读取或操作。
- 将 WebView Agent 活动提示移动到底部，并直接使用 `src/theme` 语义色变量渲染状态颜色。
- 修复点击新建会话按钮时聊天输入框不会获得焦点的问题：BChat 通过 `defineExpose` 暴露 `focusInput` 方法，ChatSider 在 `handleCreateDraftSession` 中显式调用，确保已在草稿态时仍能聚焦输入框。
- Skill 设置列表分页状态改为通过路由查询参数 `?page=` 维护，切换分页使用 `router.replace` 同步 URL；详情页返回按钮改为 `router.back()`，从技能详情返回列表时自动保留之前的页码，列表项数收缩导致越界时自动回退到最后一页。
- BChat runtime 请求仅携带已启用且配置完整的 MCP server，避免关闭或不可运行的 MCP 配置继续占用输入 Token。
- 记忆设置页进入时主动加载 `MEMORY.md`，避免首次打开时把未加载状态误显示为暂无记忆。
- 设置路由标签新增配置图标，并让 HeaderTabs 优先使用 tab 配置图标渲染顶部标签图标。
- 修复 BChat 单轮上下文压缩体验：长任务在 tail 保留策略吃空可压缩消息时仍可压缩，短内容则显示“无需压缩”友好提示。
- 修复部分模型在工具调用后以 `stop` 结束流导致长任务完成上下文压缩后不再继续执行的问题。
- 调整 BChat 自动上下文压缩展示：自动压缩改为写入当前 assistant 消息的 `compaction` part，手动压缩仍保留独立压缩消息。
- 补齐 assistant `compaction` part 的上下文恢复、气泡渲染与回滚记录清理，避免长任务压缩后中断或遗留悬空压缩记录。
- 修复 BChat 长任务续轮压缩时机：工具续轮需要继续执行且 usage 触发压缩时，先压缩再进入下一轮模型调用，避免 assistant 被完成态收口后才压缩。
- 修复 BChat 自动压缩后 provider 报 `messages` 参数非法的问题：压缩摘要改为 `system` 上下文，并在压缩覆盖 active assistant 时补充用户态续跑指令。
- 修复 BChat 自动压缩后同轮新增工具结果再次触发 `messages` 参数非法的问题：将 compaction 后的 assistant/tool 进展转为文本型续跑上下文，避免向模型发送被压缩边界截断后的结构化 tool 消息链。
- 修复 BChat 长任务多次自动压缩边界判断：同一 assistant 内 compaction 后新增进展不再误判为 already compact，新的 pending/failed/skipped 状态也不再删除上一条成功压缩边界。
- 修复 BChat 压缩完成后上下文估算不刷新的问题：自动与手动压缩成功后会重新广播基于压缩后消息的 `context-usage-updated` 快照。
