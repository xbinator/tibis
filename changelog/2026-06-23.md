# 2026-06-23

## Changed
- WebView 最近记录写入页面 favicon，并在欢迎页与最近记录搜索弹窗中优先展示站点图标，缺失或加载失败时回退到默认网页图标。
- 抽离最近记录图标为 `BRecentIcon` 组件，统一处理文件图标、WebView favicon 与默认回退图标；`addWebviewRecord` 的 favicon 入参改为 options 对象。
- 将最近记录搜索弹窗组件从 `BSearchRecent` 重命名为 `BRecent`，并将图标组件迁移到 `src/components/BRecent/Icon.vue`。
- 文件提及菜单改用 `BRecentIcon` 渲染文件图标，确保 `package.json`、`.tibis` 等特殊文件名与最近记录列表使用同一套图标逻辑。
- 聊天输入框 @ 文件提及列表仅展示 Markdown 最近文件，避免插入暂不支持的文件类型。
- 聊天对话滚动到底部按钮在远离底部且模型生成中保持可见，滚到底部时仍按滚动状态隐藏。
- BPanelSplitter 拖拽期间增加全屏透明拖拽屏蔽层，避免鼠标在 WebView 上松开后拖拽状态无法结束。
- 修复 BPanelSplitter 拖拽屏蔽层导致外部 class 无法透传的问题，恢复 ChatSider 放大态样式。
- 统一聊天上下文用量估算与预算口径到 `shared/ai/context`，前端展示与主进程发送前判断复用同一套 token 权重、输出预留和阈值。
- 将 Rich 表格内联 AI 选区的 CSS Custom Highlight 样式改为运行时注入，避免 Lightning CSS 构建警告。
- 增强 WebView Agent 可操作元素识别，补充 BrowserState 风格 `summary`、可点击置信分、推断原因、语义路径、中心点命中目标，并让点击优先派发到命中的子节点。
- 收紧 WebView Agent `navigate` 契约，仅用于用户明确给出 URL 的地址栏导航，页面可见项必须使用 `read_current_webpage` 的 `[N]` 句柄操作。
- 修复 WebView 页面短暂 loading 事件导致刚读取的 Agent 快照立即过期的问题，并避免小型固定工具条被误判为顶层浮层。
- 增强 WebView Agent 顶部短文本入口识别，支持无语义容器、无显式点击样式的紧凑工具栏文本被标记为可操作元素。
- 优化 WebView Agent `summary/content` 简化 DOM 序列化，避免父容器合并子元素文本形成误导性的单一可点击描述。
- 将 WebView 自动化的快照脚本、操作脚本、规范化逻辑和内部类型拆分到 `src/views/webview/web/automation`，降低 `useWebView.ts` 职责复杂度。
- 重构 WebView `read_current_webpage` 页面读取链路，改为共享扁平 DOM 树采集、page-agent 风格简化 DOM 输出，并让 `operate_webpage` 复用同一套元素索引规则。
