# 2026-07-17

## Added

- 为 BSkill 真实文件预览头部增加可选编辑入口，并仅在 Skill 详情页启用，以将当前选中文件交给 Markdown 编辑器打开。

## Changed

- 重构 `SkillItemRow.handleEditSkill`，改用 `asyncTo` 归一化错误处理并合并异常 / 空结果两条提示，移除手写 try/catch 和重复日志。
- 重构 `SkillItemRow.handleDeleteSkill`，按步骤使用 `asyncTo` 包裹 `trashFile` 与 `rescan`，以早返回替代 `movedToTrash` 标志位；删除锁改为 `try/finally` 维护，移除组件内冗余日志。
- 为聊天输入框和用户消息气泡中的技能引用 Chip 增加 `lucide:hammer` 图标，统一聊天 Chip 与设置列表项的 Skill 详情页导航，并在 Widget 销毁时同步释放图标应用。
- 统一用户输入引用片段的文件、技能判别值，并移除 ChipResolver 子目录代理导出入口。
- 将 BText 斜杠选项收敛为通用选中行为配置，命令与技能领域语义由 BChat 持有。
- 允许用户通过斜杠菜单显式选择禁用技能，同时保持模型自动 Skill 工具只使用已启用技能。
- 收敛 BChat SkillReference 名称校验、引用分段与 Runtime 临时上下文数据结构。
- 统一技能与工具相关图标为 lucide:hammer，替代 lucide:wrench。
- BSkill 预览头部操作按钮（编辑 / 复制）默认隐藏，hover 面板时显示，弱化非核心操作。
- Skill 详情页路径区的复制按钮默认隐藏，hover 时显示，避免干扰路径信息浏览。
- 在 AGENTS.md 中新增"沟通规范"章节（已调整至文件最顶部），要求每次回答前先说"亲"。
- 移除已无运行时消费者的通用文件会话 Hook，将文件状态类型迁移为共享契约，并收窄 Widget 子 Hook 的会话依赖。
- 以 `useFileController` 统一编辑器与 Widget 的草稿持久化、保存状态、磁盘写入、文件监听和标签脏状态，并通过 `on*` 事件保留各文件类型的解析、对话框与冲突协调能力。
- 将“关闭自动保存”明确为仅禁止自动写入磁盘，草稿仍持续保存；同时增加异步加载、另存为和自写入监听的过期结果隔离。
- 整改 `src/hooks` 目录结构：将 `useAntdTheme.ts` 改为 `useAntdTheme/index.ts`；将原 `useFileController.ts` 主体与 `file-controller/` 子模块统一收敛到 `useFileController/` 目录（`index.ts` / `types.ts` / `useDiskSave.ts` / `useDraftPersistence.ts` / `useFileWatch.ts`），并删除旧的 `file-controller/` 目录；类型从 `useFileController/types` 各自按需引入，不再由 `index.ts` 聚合再导出。
- 续整改 `src/hooks` 目录结构：将其余 17 个 hook 文件（`useAutoCollapse` / `useChat` / `useChatActorSystem` / `useChatRuntimeEvents` / `useChatRuntimeRecovery` / `useClipboard` / `useFileDrop` / `useImagePreview` / `useMenuAction` / `useNavigate` / `useOpenDraft` / `useOpenFile` / `useScroller` / `useShortcuts` / `useSystem` / `useWatchResource` / `useWorkspaceRoot`）统一改为文件夹形式（`useXxx/index.ts`），并把 `useChatRuntimeEvents` 中的无状态错误转换辅助抽离为 `errorMapping.ts`，与 `useFileController` 拆分风格保持一致；调用方 `from '@/hooks/useXxx'` 形式的 import 路径无需修改，由 TypeScript 自动解析到 `index.ts`。
- 将 `useChatActorSystem` / `useChatRuntimeEvents` / `useChatRuntimeRecovery` 三个 Chat 运行时 hook 迁移到 `src/hooks/useChat/` 下作为子目录（`useChat/useChatActorSystem/` / `useChat/useChatRuntimeEvents/` / `useChat/useChatRuntimeRecovery/`），与原 `useChat`（AI 流式生成）一起组成 Chat 领域统一目录；同步更新 `App.vue` / `BChat/index.vue` / `useChatActorSystem` 内部以及两个 chat runtime 测试文件的 import 路径。
- 进一步整理 `useChat/` 目录：去掉三个 Chat 运行时 hook 的子目录形式，改用 `useActorSystem.ts` / `useRuntimeEvents.ts` / `useRuntimeRecovery.ts` + 同级 `errorMapping.ts` 的扁平 `.ts` 结构，与 `useChat`（AI 流式）平级；同步重命名函数（`useChatActorSystem` → `useActorSystem` / `useProvideChatActorSystem` → `useProvideActorSystem` / `useChatRuntimeEvents` → `useRuntimeEvents` / `useChatRuntimeRecovery` → `useRuntimeRecovery` / `recoverChatRuntimes` → `recoverRuntimes`），并把 `use-chat-runtime-events.test.ts` / `use-chat-runtime-recovery.test.ts` 测试文件改名为 `use-runtime-events.test.ts` / `use-runtime-recovery.test.ts`。

- 将 `FileSessionState` 重命名为 `FileState` 并迁移到 `@/shared/platform/native/types`，与 `File` 接口并列；删除 `src/hooks/types.ts`，所有引用统一从 `native/types` 引入。
- 简化 `widget/editor` 文件会话的 `onResolveConflict`，去除冗余的 `asyncTo` 包裹与 `if (error) throw` 早返回，直接解构 `Modal.confirm` 的 `[cancelled, confirmed]` 元组，按 `confirmed` 判定 `useDisk` / `keepDraft`，与项目内其它 `Modal.delete/confirm` 直 await 的风格保持一致。
