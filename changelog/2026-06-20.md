# 2026-06-20

## Changed
- 将 `electron/main/modules/chat/runtime/stream-executor.mts` 拆分为 `electron/main/modules/chat/runtime/stream/` 目录，按职责划分为 `chunks.mts`、`tools.mts`、`message-parts.mts`、`request.mts`、`types.mts`，主循环逻辑合入 `index.mts`。

## Fixed
- 修复内置 AI 服务商只向 `settings.json` 写入用户覆盖字段导致直接读取配置时缺少服务商完整数据的问题。
- 修复聊天运行时发送、续轮、工具结果与 bridge 响应等 Electron IPC 请求无法克隆 Vue Proxy 导致失败的问题。
- 修复多工具调用时后续工具结果覆盖前一个工具状态的问题：改为累积 `executedToolCount`、`allToolsContinueable`、`anyToolStopped` 状态。
- 修复流式 JSON 在未闭合前 parse 失败导致 `input` 被重置为 `null` 的 UI 闪烁问题，保留上一次成功解析的值。
- 修复未注册工具调用长期处于 `executing` 状态且无反馈的问题，流结束时未收到 `tool-result` 的调用会兜底为 `TOOL_NOT_FOUND` 失败结果。
- 修复 `sourceMessages` 为空时多模态文件内容丢失的问题，统一通过 `toRuntimeModelMessages` 处理 `userMessage`。
