# 2026-05-27

## Added

- 新增依赖策略测试，防止 native `tree-sitter` 重新进入运行时依赖和 Electron rebuild 链路。

## Changed

- Release 工作流改为分别构建 macOS arm64 与 x64 产物，避免只生成单一 mac 架构包。
- 将 `dayjs` 调整为生产依赖，确保 Electron 主进程打包后可正常解析聊天服务的时间工具依赖。
- 主进程构建前自动清理 `dist-electron`，避免陈旧编译产物进入 Electron 安装包。
- 移除 Shell runner 对 `effect` 顶层入口的依赖，避免 Electron 运行时加载 `effect/FastCheck` 并缺失 `fast-check`。
- 显式声明 Tavily/axios/form-data 运行时依赖闭包，避免 Electron 打包后缺失 `es-object-atoms`。
- 移除直接依赖的 native `tree-sitter`，改为仅保留 `web-tree-sitter` 与 grammar WASM 包。
- 将 `postinstall` 收窄为 `electron-rebuild --only better-sqlite3`，避免重编译只用于 WASM 文件的 tree-sitter grammar 包。
- 通过 `pnpm.peerDependencyRules.ignoreMissing` 忽略 grammar 包的 optional `tree-sitter` peer，避免 pnpm 自动安装 native `tree-sitter`。

## Removed

- 移除 Release 工作流安装依赖阶段的 `CXXFLAGS` / `CL` 环境变量兜底，避免 Windows 上被依赖内置 `/std:c++17` 覆盖后产生误导。
