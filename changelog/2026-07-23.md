# 2026-07-23

## Added

- 欢迎页新增独立聊天入口，并支持从最近记录打开已存在的聊天会话。
- 新增 `useRecentRecord` 公共用例，统一最近记录的打开、陈旧聊天清理与删除行为。
- 新增柔和黑白「Graphite」主题预设，模拟白色工作区与浅灰产品外壳的界面层级。

## Changed

- 补充 Primary 自动委派 Child Agent 设计，新增 Primary 挂起与结果汇合、Child 专用执行入口、工具副作用分类、并行确认队列和结果证据验证协议。
- 新增 Child Agent 持久化委派基础实施计划，按运行时地址、不可变存储、挂起栅栏、结果汇合和 Actor 恢复拆分首个交付阶段。
- 重构最近记录模型：文件、Widget、WebView 与聊天统一包含 `type`、`id`、`url`、`title` 等展示与跳转字段，最近记录打开入口优先按 `url` 路由。
- 将聊天最近记录的业务 ID 收敛为会话 ID，使用 `type + id` 作为稳定键，并兼容清理旧的 `chat:{sessionId}` 记录形态。
- 补齐拖拽草稿、编辑器草稿、Widget 草稿和新建文件的统一最近记录字段，减少新增记录类型时需要修改的分支。
- 欢迎页与命令面板改为复用统一最近记录动作，页面层不再各自维护文件、聊天与 WebView 打开分支。
- 将最近记录打开/删除逻辑改为 `Record<RecentRecordType, RecentRecordHandler>` 处理器表，新增记录类型时由类型检查提示补齐处理器。
- 标签页最近记录图标改用 `type + id` 稳定键匹配，避免文件与 Widget 共用业务 ID 时串用记录。
- 抽出最近记录展示派生函数，欢迎页和命令面板不再直接维护文件型记录展示分支。
- 最近记录存储归一化改为 `Record<RecentRecordType, RecentNormalizer>` 处理器表，保留旧文件记录迁移兼容并减少新增类型时的分支修改。
- 标签页模型新增 `recentKey` 字段，HeaderTab 不再解析 editor/widget/chat 路由来猜最近记录身份，旧持久化标签在 store 归一化时自动补齐。
- 抽出 `useHeaderTabIcon` 收口 HeaderTab 图标属性解析，组件层不再维护最近记录索引与 WebView URL 匹配细节。
- 将 `useNavigate` 拆分为 link、document、reference、draft、skill 子模块，统一承接原文件打开、草稿创建、引用定位与页面跳转能力。
- 将 `useNavigate` 的具体动作模块收纳到 `actions/` 目录，根目录只保留门面入口与类型定义。
- 最近文档记录打开能力在 `useNavigate` 中改名为 `openDocument(record)`，保留 `openFile(options)` 专用于聊天/富文本引用定位。
- 删除聊天会话后的路由同步会一并移除聊天最近记录，最近聊天记录打开时不再重复加载会话做存在性校验。
- 优化 SettingsTool 的主题预设说明，将泛泛的“主题色”改为整套界面色彩氛围语义，并补充默认、Graphite、Manga Ink 与 Shonen 的主色描述。

## Fixed

- 修复 BSmart 文件提及菜单图标解析只使用文件名主体的问题，并将文件名与扩展名组合逻辑收敛到 `BRecentIcon` 内部。
- 修复独立聊天页打开或从 KeepAlive 标签切回时未聚焦底部智能输入编辑器的问题。
- 修复从聊天侧栏打开或切换到独立聊天页后焦点仍回到侧栏输入框的问题，已激活的独立聊天页也会响应打开与会话切换请求重新聚焦输入框。

## Removed

- 移除晴空蓝「Velora」主题预设，主题列表不再注册 `velora`。
- 移除命令面板最近记录 source 对文件与聊天的硬编码打开分支，改由统一 `openRecent(record)` 入口处理。
- 移除最近记录相关源码结构断言测试，改由类型检查和行为测试覆盖，降低测试维护心智负担。
- 移除 `useOpenFile`、`useOpenDraft` 和 `useRecentRecordActions` 独立 hook 入口，调用方统一改用 `useNavigate` 与 `useRecentRecord`。
