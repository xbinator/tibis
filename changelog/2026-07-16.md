# 2026-07-16

## Added

- 新增 BChat 显式 SkillReference：斜杠菜单按命令与技能分组，支持在正文任意空白边界插入结构化技能引用，并由 Chat Runtime 按当前最新内容生成仅本轮生效的用户级技能上下文快照。
- 新增 BChat 长会话上下文压缩设计文档，明确 80% 同步触发、结构化滚动 checkpoint、稳定 artifact 身份、Part 拓扑校验、并发写入租约和用户可见状态方案。
- 新增 BChat 长会话上下文压缩实现计划，按共享契约、预算、拓扑、指纹、artifact、摘要、投影、Runtime、手动命令、UI 与质量回归拆分 TDD 任务。
- 新增 BChat 长会话上下文压缩：模型请求前达到约 80% 输入预算时同步压缩到约 55% 目标，并支持空闲会话通过 `/compact` 手动触发。
- 新增消息流内压缩状态 Part，仅展示“压缩中、已压缩、失败、已取消、无需压缩”五种用户状态，不展示摘要、Token、比例或诊断字段。
- 新增输入工具栏上下文用量环形入口，悬浮展示当前模型输入投影的已用 Token、上下文窗口和使用比例，并在模型续轮及压缩完成后实时刷新。
- 新增 AI Runtime 可靠性收尾设计、Node 22 本地版本文件，以及不可配置的工具循环和模型请求超时策略。
- 新增 main 与 pull request 的 Node 22 质量门禁，覆盖 renderer/Electron 类型检查、ESLint、Stylelint 和完整测试。

## Removed

- 完整移除项目上下文中消息压缩（context compaction）功能：
  - **renderer**：删除 `useContextUsage` hook、`useUsagePanel` hook、`useRuntimeCompactContext` hook、`UsagePanel` 组件、`ContextUsage` 组件、`tokenEstimator` 工具函数、`compression` 相关类型与常量
  - **主进程**：删除 `compactionService`、`compactionExecutor`、`structuredSummaryGenerator`、`contextBudgetService`、`tokenEstimator`、`contextOverflow` 等压缩相关模块，移除 `ChatRuntimeCompactInput/Result` 类型与 IPC `compact` 方法
  - **状态机**：从 `sessionMachine` 移除 `compacting` 状态与 `compactRequested/compactCompleted/compactFailed` 事件
  - **类型**：移除 `ChatMessageCompactionPart`、`ChatMessageRuntimeMeta`、`ChatCompressionStatus`、`ChatCompressionMeta`、`ChatRuntimeCompactInput`、`ChatRuntimeContextUsageSnapshot/Event`、`CompressionRecord/Status` 等类型定义；`ChatMessageRole` 移除 `'compression'`；`ChatRuntimeEventName` 移除 `'chat:runtime:context-usage-updated'`
  - **数据库**：移除 `chat_messages` 表 `compression_json` 列、`meta_json` 列、`summary` 列；移除 `chat_session_compression_records` 表
  - **IPC**：移除 `chatCompression*` 系列与 `chatRuntimeCompact`、`chatRuntimeOnContextUsageUpdated` API
  - **测试**：删除所有压缩相关测试文件与测试用例；更新受影响测试的角色排序期望值（`compression` 角色移除导致 `user` 排序权重从 2 变为 1）
- 移除 AI SDK 6 的 `tool-input-available`、工具输入字段和工具结果字段兼容路径，流式边界仅接受 AI SDK 7 事件结构。

## Changed

- BChat 技能引用标记统一为 `{{$技能名}}`：斜杠菜单和输入框不再展示技能图标，输入框与用户消息均只展示技能名；消息 `content` 保留可逆 Token，并继续通过结构化 `skill_reference` part 驱动 Runtime。
- BChat 引用解析器按 `chipResolver/file/` 与 `chipResolver/skill/` 重组，文件引用的展示模型、DOM 工具与样式从组件目录迁入对应解析器目录。
- `todowrite` 工具消息改为单层任务卡片，顶部直接展示完成进度，列表仅保留原有状态图标与任务文本；任务基础结构继续与 `TodoPanel` 共用。
- `todowrite` 工具气泡标题尾部追加 `已完成数/总数` 进度标签，与 `TodoPanel` header 视觉风格一致，仅在工具结果为成功且存在任务快照时展示。
- `TodoPanel` 抽出 `todo-list` 卡片外壳，header 与可折叠列表共享同一容器；`TodoList` 仅渲染任务条目 body，保持 `todowrite` 单层任务卡片的形态。
- `TodoList` 完成状态的任务文本以划线展示，与取消状态保持一致的删除线视觉。
- ChatSider 当前会话标题支持双击进入 AInput 编辑，通过 `v-focus` 自动聚焦并全选，回车或失焦后持久化标题并刷新会话历史；标题编辑状态统一收拢到单一响应式对象。
- 独立会话分支标题使用全角括号序号：首次分支追加 `（2）`，从已有分支继续创建时递增末尾数字。
- 上下文用量入口仅在已取得有效用量时显示；入口默认使用透明背景，仅在鼠标移入时显示背景，将可见圆环收窄为 16px，使用应用主题色展示进度，并保持用量 Tooltip 单行显示。
- Node.js 开发文档和本地版本文件保留最低支持版本 22，GitHub Actions 发布构建使用 Node 24 LTS；AI SDK 完整事件流恢复强类型，并同时暴露末步 `stepUsage` 与调用累计 `totalUsage`。
- 主聊天工具续轮统一由 Chat Runtime 托管，Renderer、本地主进程、Tavily 与 MCP 不再限制工具执行次数，仅保留 300 秒任务级上限和连续重复调用收口；`finishReason = stop` 不再触发多余续轮。
- AI SDK 7 的工具异常、输出拒绝、审批拒绝和流中止事件改为显式终态，并以穷举事件分支防止后续 SDK 新事件被静默忽略。
- 上下文压缩消息统一复用 `BubblePartStatus` 状态组件，并移除独立的 `BubblePartCompaction` 组件，保持中断与压缩状态的消息流样式一致。
- `executeRuntimeStreamRounds` 中移除 `compactSourceMessagesForUsageIfNeeded` 调用，改为调用纯 `createContinuationSourceMessages` 将 assistant 草稿纳入续轮上下文
- `runRuntimeStream` 移除 `compactBeforeStreamIfNeeded`、`compactAfterProviderUsageIfNeeded`、`replayAfterContextOverflow` 调用链路
- `branch.mts` 保留 `branchSession`/`createSessionBranchData` 核心逻辑，仅剥离压缩记录克隆与 `ChatMessageRuntimeMeta` 引用
- 上下文投影改为使用最新有效结构化 checkpoint 与原始 tail；大型旧工具结果只在模型投影中软裁剪，数据库原文保持不变。
- 长 Agent 任务达到压缩阈值后会优先裁剪当前轮次中较早的完整大型工具结果；最新结果单独导致预算不足时再裁剪其正文，但保留 Part 拓扑、调用信息和 artifact identity。摘要容量据此重新规划，数据库原文保持不变。
- 压缩期间继续允许编辑输入草稿，但分支、回滚和重新生成在组件与事件处理两层禁用，避免通过历史操作中断 pending checkpoint。
- 消息气泡不再接收会话运行状态，分支、回滚和重新生成统一由上层事件入口阻止忙碌期间的历史写操作。
- 文件工具结果与结构化摘要使用稳定 artifact identity，支持文件移动后继续关联同一产物，并区分删除后同路径新建的产物。
- 上下文 Token 估算采用更保守的跨 Provider 字符权重，并按摘要生成器实际序列化 prompt 计算输入预算，避免 Unicode 或大量小 Part 导致延迟触发。
- 将 `useChatSessionLifecycle` 中自动命名会话镜像 `currentSessionForAutoName` 重命名为 `autoNameSession`，与 `scheduleAutoName`/`captureAutoNameSnapshot` 命名风格对齐；移除 `BChat/index.vue` 中未使用的同名解构以消除 ESLint `no-unused-vars` 警告。

## Fixed

- 修复兼容模式 Provider 可能返回不满足结构化摘要约束而导致压缩直接失败的问题：首次校验失败时按脱敏子错误码自动修复重试一次，仍失败则在 checkpoint 与诊断日志保留安全子错误码，不保存或展示无效原始输出；同步升级压缩策略版本，避免旧失败指纹阻止新版自动重试。
- 修复 AI SDK v7 合并校验发现的类型检查残留：移除已由 Actor Runtime 注册表替代的 renderer 回退忽略缓存，并将 BText 滚动测试夹具同步为现有 `/compact` 命令。
- **测试修复**：补全上下文压缩功能移除后遗留的测试问题，使全部用例通过：
  - `src/utils/asyncTo.ts`：在 `asyncTo` 中对 `logger.error` 调用加 `try/catch`，避免 `cause` 为已吊销 Proxy 时 `console.error` 调用 `getPrototypeOf` 二次抛错，保证错误归一化函数永不 reject
  - 补全 `use-auto-name.test.ts`、`skill-creator.test.ts`、`settings-button.test.ts` 中 `@/shared/platform/electron-api` mock 缺失的 `hasElectronAPI` 导出（置为 `() => false`，走 console 日志分支）
  - 补全 `widget/index.test.ts` 中 `@/utils/logger` mock 缺失的命名导出 `logger`
  - `ipc.test.ts`：首个用例加 `win32` 平台守卫（正斜杠路径规范化仅 Windows 有意义），与同文件第二个用例保持一致
  - 删除 `use-usage-panel.test.ts`、`context-estimator.test.ts`、`usage-panel.test.ts` 三个孤儿测试（源码 `useUsagePanel`、`UsagePanel` 与上下文压缩估算器已在本次重构中删除）
- 修复会话分支、回滚和重新生成后的 checkpoint 依赖安全：分支重写 Part 引用并按保存快照重算 fingerprint，完整消息截断写库前移除失效 checkpoint 链。
- 修复应用重启后遗留 pending checkpoint 无法收敛的问题，恢复时在会话写锁下标记为 `INTERRUPTED`，保留既有成功 checkpoint。
- 修复 AI 服务日志可能记录完整压缩 prompt、工具正文和结构化摘要的问题，改为仅记录请求标识、模型、功能开关与 Token usage 白名单字段。
- 修复 pending 首次写入失败、取消期间提前释放会话锁和启动恢复失败不再重试的问题，保证 checkpoint 在写锁释放前收敛到稳定终态。
- 修复 `read_current_document` 缺少稳定 `artifactId` 的问题，使当前文档与后续文件操作可复用同一产物身份。
- 修复工具循环只因单个重叠调用就误判重复的问题，改为比较相邻步骤的完整无序调用集合；收口阶段同时阻止内部工具协议作为普通文本泄漏。
- 修复工具超时仅结束等待但不发送中止信号的问题，工具执行器、Renderer 请求和主进程长文件搜索现会共享工具级中止信号。
- 修复 AI SDK 中止信号未传入 MCP 工具调用、MCP discovery 不受请求截止时间约束，以及自动上下文压缩模型调用可能重新获得完整 300 秒的问题。
- 修复 AI SDK 7 用户图片仍使用旧 `image` part 的问题，统一转换为带 URL 数据和媒体类型的 `file` part。
- 修复 `todowrite` 空任务列表缺少提示、组件传入无效属性及回归测试仍断言旧进度位置的问题。
- 修复 `atomically` 失败清理测试在高并发负载下偶发早于临时文件删除完成的问题，质量门禁会在有限时间内等待清理收敛。
