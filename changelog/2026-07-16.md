# 2026-07-16

## Added

- 新增 BChat 长会话上下文压缩设计文档，明确 80% 同步触发、结构化滚动 checkpoint、稳定 artifact 身份、Part 拓扑校验、并发写入租约和用户可见状态方案。
- 新增 BChat 长会话上下文压缩实现计划，按共享契约、预算、拓扑、指纹、artifact、摘要、投影、Runtime、手动命令、UI 与质量回归拆分 TDD 任务。
- 新增 BChat 长会话上下文压缩：模型请求前达到约 80% 输入预算时同步压缩到约 55% 目标，并支持空闲会话通过 `/compact` 手动触发。
- 新增消息流内压缩状态 Part，仅展示“压缩中、已压缩、失败、已取消、无需压缩”五种用户状态，不展示摘要、Token、比例或诊断字段。

## Removed

- 完整移除项目上下文中消息压缩（context compaction）功能：
  - **renderer**：删除 `useContextUsage` hook、`useUsagePanel` hook、`useRuntimeCompactContext` hook、`UsagePanel` 组件、`ContextUsage` 组件、`tokenEstimator` 工具函数、`compression` 相关类型与常量
  - **主进程**：删除 `compactionService`、`compactionExecutor`、`structuredSummaryGenerator`、`contextBudgetService`、`tokenEstimator`、`contextOverflow` 等压缩相关模块，移除 `ChatRuntimeCompactInput/Result` 类型与 IPC `compact` 方法
  - **状态机**：从 `sessionMachine` 移除 `compacting` 状态与 `compactRequested/compactCompleted/compactFailed` 事件
  - **类型**：移除 `ChatMessageCompactionPart`、`ChatMessageRuntimeMeta`、`ChatCompressionStatus`、`ChatCompressionMeta`、`ChatRuntimeCompactInput`、`ChatRuntimeContextUsageSnapshot/Event`、`CompressionRecord/Status` 等类型定义；`ChatMessageRole` 移除 `'compression'`；`ChatRuntimeEventName` 移除 `'chat:runtime:context-usage-updated'`
  - **数据库**：移除 `chat_messages` 表 `compression_json` 列、`meta_json` 列、`summary` 列；移除 `chat_session_compression_records` 表
  - **IPC**：移除 `chatCompression*` 系列与 `chatRuntimeCompact`、`chatRuntimeOnContextUsageUpdated` API
  - **测试**：删除所有压缩相关测试文件与测试用例；更新受影响测试的角色排序期望值（`compression` 角色移除导致 `user` 排序权重从 2 变为 1）

## Changed

- 上下文压缩消息统一复用 `BubblePartStatus` 状态组件，并移除独立的 `BubblePartCompaction` 组件，保持中断与压缩状态的消息流样式一致。
- `executeRuntimeStreamRounds` 中移除 `compactSourceMessagesForUsageIfNeeded` 调用，改为调用纯 `createContinuationSourceMessages` 将 assistant 草稿纳入续轮上下文
- `runRuntimeStream` 移除 `compactBeforeStreamIfNeeded`、`compactAfterProviderUsageIfNeeded`、`replayAfterContextOverflow` 调用链路
- `branch.mts` 保留 `branchSession`/`createSessionBranchData` 核心逻辑，仅剥离压缩记录克隆与 `ChatMessageRuntimeMeta` 引用
- 上下文投影改为使用最新有效结构化 checkpoint 与原始 tail；大型旧工具结果只在模型投影中软裁剪，数据库原文保持不变。
- 文件工具结果与结构化摘要使用稳定 artifact identity，支持文件移动后继续关联同一产物，并区分删除后同路径新建的产物。
- 上下文 Token 估算采用更保守的跨 Provider 字符权重，并按摘要生成器实际序列化 prompt 计算输入预算，避免 Unicode 或大量小 Part 导致延迟触发。
- 将 `useChatSessionLifecycle` 中自动命名会话镜像 `currentSessionForAutoName` 重命名为 `autoNameSession`，与 `scheduleAutoName`/`captureAutoNameSnapshot` 命名风格对齐；移除 `BChat/index.vue` 中未使用的同名解构以消除 ESLint `no-unused-vars` 警告。

  - 补全 `use-auto-name.test.ts`、`skill-creator.test.ts`、`settings-button.test.ts` 中 `@/shared/platform/electron-api` mock 缺失的 `hasElectronAPI` 导出（置为 `() => false`，走 console 日志分支）

  - `ipc.test.ts`：首个用例加 `win32` 平台守卫（正斜杠路径规范化仅 Windows 有意义），与同文件第二个用例保持一致
