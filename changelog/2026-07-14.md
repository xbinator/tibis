# 2026-07-14

## Added

- BChat 助手消息工具栏新增“创建分支”操作，可复制截至目标回答的完整会话历史并切换到标题不变的独立会话。

## Changed

- 将 `ai` 升级至 v7，并同步升级全部直接使用的 `@ai-sdk/*` 依赖；迁移系统指令、工具循环停止条件、完整事件流与 Google provider API，显式保留内部压缩 system 消息和末步 token usage 的既有语义，同时声明 Node.js 22 最低运行版本。
- `asyncTo` 将 Promise 拒绝值安全归一化为包含 `message`、`code` 与原始 `cause` 的真实错误对象，并保留 Electron API 失败结果的错误码，避免异常属性读取或错误解包破坏统一契约。
- 会话分支按钮移除请求 loading/disabled 绑定，由 BChat 底层统一防重复并通过 `asyncTo` 处理 Store 结果；分支数据重建模块归入主进程聊天 Runtime。
- 会话分支写入改用冲突即回滚的普通插入，并对源消息、压缩记录 JSON 与新生成 ID 执行严格校验，避免损坏历史或 ID 碰撞静默覆盖数据。
- 重构 AI 文件写入语义：`write_file` 与 `edit_file` 对真实路径直接原子写入磁盘，`write_file` 自动创建缺失的父目录，`unsaved://` 草稿继续通过编辑器 bridge 更新。
- 重新描述 `read_file`、`read_directory`、`write_file`、`edit_file` 的使用边界，并将 `read_file.path` 标记为必填参数。
- 增加真实文件确认后的版本复核与路径级串行写入，文件在确认期间变化时返回冲突，避免覆盖新内容或重新创建已删除的编辑目标。
- 写入前按 `realpath` 复核符号链接真实目的地，工作区外写入使用危险级确认并展示真实路径；补充 Windows 路径和原子失败清理测试。
- BChat Composer 直接接收输入编辑器实例引用与交互 API，并复用 BTextEditor 暴露方法类型，清理入口组件的编辑器方法、Toast 方法与清空草稿转发。

## Fixed

- BChat 将消息持久化收口到 Chat Workflow，并避免异步创建分支完成后抢回用户已切换的会话；同步清理入口组件类型抑制与函数签名。
- Widget 编辑器禁止重命名固定的 `widget.json` 配置文件，避免已安装 Widget 脱离扫描路径。
- Widget 编辑器“另存为”改为仅导出副本，不再将当前安装会话绑定到导出路径。
- Widget 文件删除后重新创建时同步读取最新内容，并复用外部修改的草稿协调流程。
- `developing-widget` Skill 新增 Widget 安装路径铁律：必须写入 `~/.tibis/widgets/<id>/widget.json`（复数 `widgets/`），禁止写入 `~/.tibis/<id>/widget.json` 或 `~/.tibis/widget/<id>/widget.json`；`references/widget-format.md` 新增 `Install Location` 章节说明。
- 修正 `developing-widget` Skill 中 validator 命令的路径：原 `docs/development/developing-widget/scripts/validate-widget.js` 不存在，实际位置是 `skills/developing-widget/scripts/validate-widget.js`；同步按"当前文件相对目录"语义改写为 `scripts/validate-widget.js`（SKILL.md）与 `../scripts/validate-widget.js`（references/widget-format.md）。
- 同步 `developing-widget` Skill 模板 `widget.json` 的 `execute.code` 与实际默认生成器（`createWidgetScriptDefaultCode`）保持一致：class 名 `Component`、同步 `onExecute()`、新增 `onMounted()`、补 JSDoc。
- 补全 `references/elements-and-bindings.md` 三处说明：`loop.autoColumns: true` 时 `columns` 可写 `"auto"`；`image.fit` 列出全部 5 个枚举（`cover`/`contain`/`fill`/`none`/`scale-down`）；`text.maxLines` 行为（正整数封顶，缺省/非正不限制，`\n` 计入行数）。
- 润色 `developing-widget` Skill 四处细节：frontmatter `description` 补"installing"；`widget-format.md` 的 `code` 示例从 `"…Widget {}\n"` 改为可读多行 `async onExecute() {}`；`runtime-api.md` 显式声明 `Widget` 是宿主注入全局符号（禁止 import / 重声明）；`elements-and-bindings.md` 补充子路径绑定运行时解析但不参与 schema 静态校验的说明。
- 修复 `developing-widget` Skill validator 脚本 ESLint 全部 12 个问题：3 个 `no-await-in-loop`（`scanPackageFiles.visitDirectory` 与 `validateLocalResources` 改为 `Promise.all` 并行）；8 个 `no-use-before-define`（重排函数声明：`warnWhenElementExitsCanvas`、`validateButtonAction` / `validateButtonMetadata`、`validateGroupChildren`、`formatUnknownError` / `normalizePackagePath`、`collectImageSources` / `shouldSkipLocalResource` 全部前移）；1 个相互递归（`validateGroupChildren` ↔ `validateElement`）保留行级 `eslint-disable` 注释。验证：`pnpm exec eslint skills/developing-widget/scripts/validate-widget.js` exit 0；模板与故意写错的样例均按预期通过/告警。
- 修复 Widget 设置页删除小组件报 `Error invoking remote method 'ui:trashFile': Error: Failed to parse path`：`ui:trashFile` IPC handler 在调用 `shell.trashItem` 前用 `path.normalize` 将业务层传入的统一 `/` 分隔符路径转换为平台原生分隔符，避免 Windows 上 `SHFileOperation` 因正斜杠路径报 `ERROR_INVALID_NAME`；同时为 `WidgetDefinition` 补齐 `dirPath` 字段（与 `SkillDefinition` 对齐），由 parser 在解析时按 `filePath` 推导，业务层（`WidgetItemRow.vue` 等）直接消费 `widget.dirPath` 即可，删除 `getWidgetDirectoryPath` 这类临时桥接函数，业务层无需再做平台适配。
- 抽取 `src/utils/file/posix.ts` 的 `posix` 命名空间工具（仿 Node `path.posix` API），行为对齐 `path.posix.dirname`：兼容 `\`/`/` 混合输入、输出统一为 `/`。`src/ai/widget/parser.ts` 与 `src/ai/skill/parser.ts` 原各有一份重复的 `dirname` 现统一改为 `posix.dirname(...)`，后续扩展（`basename` / `extname` / `join` / `normalize` 等）只需在该文件 `posix` 对象上追加方法即可。配套单测 `test/utils/file/posix.test.ts` 覆盖 POSIX / Windows 反斜杠 / 根级 / 裸文件名 / 空串 5 种典型输入。
