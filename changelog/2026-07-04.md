# 2026-07-04

## Removed
- 删除 BWidget `constants/style.ts` 中未被引用的常量 `WIDGET_DEFAULT_PRESET_COLORS` 和 `WIDGET_COLOR_PICKER_DEFAULT_PLACEMENT`，避免冗余定义。
- 移除 BWidget 默认 JS 脚本 `WIDGET_INTERACTION_SCRIPT_DEFAULT_CODE` 中 `data` 块的 `message: ''` 示例字段，data 默认改为空对象，仅保留注释提示。同步删除 `use-element-variables.test.ts` 与 `page-setter.test.ts` 中针对该字段的断言。
- 删除 `WidgetData.viewport` 外部持久化字段，视口仅保留在 BWidget 内部编辑状态，打开时按内容重新适配。
- 移除 BWidget 运行态节点 `submit`/`widget_result` 新提交通道，运行态交互统一通过 `$sendMessage` 上行消息。

## Added
- 新增 `BText/Input.vue` 单行变量补全输入组件，支持变量按钮插入与 `{{` 触发过滤补全，复用现有变量树选择菜单。
- BWidget 图片元素地址 Setter 改用 `BTextInput`，图片 URL 字段支持从 Widget 入参与运行数据中选择变量插值。
- 新增 BWidget `useWidgetContext` 通用上下文 hook，通过 provide/inject 向 Setter 链路透传当前 Widget 数据与选中元素 ID。
- 调整 `useElementVariables` 为内部读取 `useWidgetContext`，元素 Setter 无需手动传入 Widget 数据上下文即可生成变量候选。
- 新增 Widget 文本元素「最大行数」配置项：在 Setter「显示」区块通过 BSectionItem + AInputNumber 设置，超出指定行数自动截断显示并补省略号；手动换行（\n）也计入行数配额，留空表示不限制。
- 新增 `test/components/BWidget/widget-text-metrics.test.ts`：覆盖 measureWidgetTextElementSize 的 maxLines 截断逻辑（高度收缩、宽度按可见行计算、非正值视为不限）。
- 在 `test/components/BWidget/text-setter.component.test.ts` 补充 BSectionItem / AInputNumber stub 及 5 个 maxLines 交互用例（写入/清空/初始化/默认空/保留无关元数据）。
- 新增 BWidget 图片元素（Image）：三文件结构（schema.ts + index.vue + Setter.vue）注册到 elements/index.ts，支持 URL/变量插值地址、5 种 object-fit 填充模式（cover/contain/fill/none/scale-down）、替代文本、空地址与加载失败占位视图。新增测试用例覆盖 Setter 字段写入、fit 下拉状态、视图渲染、变量插值与占位逻辑。
- 调整 `component-file-names.test.ts` 中元素 index.vue 数量断言为 3，匹配新增 Image 元素目录。
- BSectionItem 新增 `tooltip` 属性：传入后 label 区域用 `ATooltip` 包裹展示提示文本，并启用 `cursor: help` 与 hover 虚线下划线样式，未传时保持原有行为不变。
- BSectionItem 改用 TSX + `defineComponent` 实现，与 BButton 风格保持一致，保留 props 类型与插槽能力。
- BSectionBlock 新增 `tooltip` 属性：传入后标题区域用 `ATooltip` 包裹展示提示文本，并启用 `cursor: help` 与 hover 虚线下划线样式，未传时保持原有行为不变。
- BSectionBlock 新增 `tips` 属性：传入后标题右侧展示 `lucide:help-circle` 问号图标，鼠标移入通过 `ATooltip` 展示提示文本，未传时不展示图标。

## Fixed
- 修复 `BTextInput` 通过变量按钮打开下拉菜单时使用全局 Teleport 定位导致弹层跑到页面左侧的问题，改为输入框内联下拉并保持宽度一致。
- 代码审查修复：text-setter 测试补齐 BSectionItem stub，消除 `Failed to resolve component: BSectionItem` 警告。
- 代码审查修复：清理 `useElementVariables` 旧命名和过期注释，统一使用 `widgetData` 语义。
- 修复 Widget 右侧属性面板编辑已有元素内容时，BWidget 内部模型重置丢失当前视口导致画布跳动的问题。
- 修复 Widget 文本 Setter 在循环数据源未指向数组时仍展示 `item/index` 变量候选的问题。
- 调整 Widget 文本 Setter 的入参变量候选为 `$input` 展示、插入与解析，旧 `input` 模板根不再兼容。
- 调整提示词变量选择器的 `$input` 根变量右侧说明为“入参”，左侧仍显示 `$input`。
- 同步 BWidget 矩形元素组件测试选择器，匹配当前 `.widget-rect-element` 类名。
- 修复 Widget 快照入口未校验顶层 `loop` 导致非法当前模型元素被接收、后续直接读取 `element.loop` 可能异常的问题。
- 修复 Widget 元素变量候选在嵌套循环中优先使用外层循环配置的问题，改为使用离当前元素最近的循环上下文。
- 修复 Widget 运行态父级循环展开时吞掉子级循环的问题，父循环每次迭代内会继续展开子树循环并继承局部变量。
- 修复 Widget 画布模型同步回写时未保留 `execute`，导致 Ctrl+S 保存后组件脚本丢失的问题。

## Changed
- 将 `src/components/BPromptEditor` 重命名为 `src/components/BText`，原 `index.vue` 拆为 `Editor.vue` 并以 `BTextEditor` 使用，变量选择、文件提及和 Slash 命令能力随目录迁移。
- `src/components/BText/Input.vue` 原生 `<input>` 替换为 `ant-design-vue` 的 `AInput`，变量按钮移入 `suffix` 插槽，移除自定义输入框样式并采用 AInput 默认外观；同时移除 `allowClear` 属性与清空按钮，变量补全、光标同步与下拉交互行为保持不变。
- Widget 高级循环设置改为只编辑当前单选元素，移除 AdvancedSetter 内部从多选列表推断主元素的逻辑，并在多选面板隐藏高级循环页。
- Widget 高级循环设置改为和 DesignSetter 一样直接通过 `v-model` 编辑 `element.loop` 字段，移除额外字段模型。
- Widget 设置面板高级页签改为直接使用 `v-model:element="select"`，移除 `selectedTargetElement` 和 `updateAdvancedElement` 中转逻辑。
- Widget 设置面板改为通过 `useWidgetContext` 向 Text/Rect/Advanced/Batch Setter 提供只读数据，减少读取型 props 层层透传。
- Widget 元素实例模型 `WidgetShapeElement` / `WidgetElement` / `WidgetAddShapeOptions` 增加 metadata 泛型，并与 `WidgetElementSchema` 的渲染尺寸配置对齐，Text/Image 元素可按专属 metadata 类型收窄。
- Widget 元素注册配置 WidgetElementSchema 改为泛型接口 `WidgetElementSchema<TMetadata extends WidgetMetadata = WidgetMetadata>`，metadata 字段类型随之参数化；默认泛型保证读取处（WIDGET_ELEMENT_SCHEMAS、getWidgetElementSchema 返回值等）无需改动，各元素 schema 定义处可声明具体泛型获得写入端类型校验。textElementSchema 已改为 `WidgetElementSchema<WidgetTextElementMetadata>` 并移除冗余 satisfies。
- Widget 文本元素尺寸测量（measureWidgetTextElementSize / createWidgetTextRenderSize）联动最大行数配置：超出 maxLines 的行不再参与 content 尺寸计算，避免文本被 CSS 截断后元素 box 仍按全部行算高度导致下方留白。
- Widget 元素循环配置从 metadata 提升为顶层 `loop` 字段，新建元素与组合时写入默认 loop，运行态、变量候选和高级设置统一直接读取 `element.loop`。
- 删除 BWidget 旧元素兼容路径：不再兼容 `kind/shape/text/description` 旧快照字段、`borderColorWidth` 旧样式字段和运行时 `defineConfig` 空函数。
- Widget Image/Text 相关测试按当前组件实现更新：非法图片 fit 不强制回退，文本最大行数清空保留 AInputNumber 当前写入值。
- 修复 Widget 文件加载时内部模型归一化回写导致首次进入即提示未保存的问题。
- 修复 Widget 父组合缩放预览时深层子节点只跟随位置、不同步缩放尺寸的问题。
- 修复 Widget 设置面板在元素模型更新后仍编辑旧元素引用的问题。
- 修复 Widget 设置页打开组件时优先使用缓存数据，导致保存成功后再次打开仍显示旧数据的问题。
- 修复 Widget 删除持久化视口后，空数据先挂载再异步加载文件内容时不会重新适配已有元素视口的问题。
- 修复 Widget 侧栏图层拖拽到最后一个子节点后无法移出到父级同层的问题，并在同层没有可展开组合时隐藏折叠占位。
- 修复 Widget 侧栏中组外图层拖到组合最后一个子节点下方时，投放提示线错误退到父级层级的问题。
- 修复 Widget 侧栏中组外图层拖到组合第一个子节点上方时未能作为组合顶部子节点插入的问题。
- 修复 Widget 侧栏中拖拽投放线命中组合但普通列表顺序未变化时，移动事件被公共拖拽组件吞掉导致无法移入组合的问题。
- 修复 Widget 新建组合时标题使用随机元素 ID 的问题，改为按现有组合标题递增生成“组合1”等名称。
- Widget 运行态交互脚本执行完成后统一进入 `finished` 状态；未调用 `$sendMessage` 时只结束运行态并写回数据，不再发送模型消息。
