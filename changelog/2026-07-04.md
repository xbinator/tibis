# 2026-07-04

## Added
- 新增 BWidget `useWidgetContext` 通用上下文 hook，通过 provide/inject 向 Setter 链路透传当前 Widget 数据与选中元素 ID。
- 调整 `useElementVariables` 为内部读取 `useWidgetContext`，元素 Setter 无需手动传入 Widget 数据上下文即可生成变量候选。
- 新增 Widget 文本元素「最大行数」配置项：在 Setter「显示」区块通过 BSectionItem + AInputNumber 设置，超出指定行数自动截断显示并补省略号；手动换行（\n）也计入行数配额，留空表示不限制。
- 新增 `test/components/BWidget/widget-text-metrics.test.ts`：覆盖 readTextElementMaxLines 边界值归一化（0/负数/NaN/Infinity/字符串/小数）与 measureWidgetTextElementSize 的 maxLines 截断逻辑（高度收缩、宽度按可见行计算、非正值视为不限）。
- 在 `test/components/BWidget/text-setter.component.test.ts` 补充 BSectionItem / AInputNumber stub 及 5 个 maxLines 交互用例（写入/清空/初始化/默认空/保留无关元数据）。
- 新增 BWidget 图片元素（Image）：三文件结构（schema.ts + index.vue + Setter.vue）注册到 elements/index.ts，支持 URL/变量插值地址、5 种 object-fit 填充模式（cover/contain/fill/none/scale-down）、替代文本、空地址与加载失败占位视图。新增 12 个测试用例覆盖 Setter 字段写入、fit 归一化、视图渲染、变量插值与占位逻辑。
- 调整 `component-file-names.test.ts` 中元素 index.vue 数量断言为 3，匹配新增 Image 元素目录。

## Fixed
- 代码审查修复：Text Setter computed set 类型签名补齐 `null`（AInputNumber allow-clear 清空时传入 null），显式处理 null 分支避免类型不严谨。
- 代码审查修复：text-setter 测试补齐 BSectionItem stub，消除 `Failed to resolve component: BSectionItem` 警告。
- 代码审查修复：清理 `useElementVariables` 旧命名和过期注释，统一使用 `widgetData` 语义。

## Changed
- Widget 高级循环设置改为只编辑当前单选元素，移除 AdvancedSetter 内部从多选列表推断主元素的逻辑，并在多选面板隐藏高级循环页。
- Widget 设置面板改为通过 `useWidgetContext` 向 Text/Rect/Advanced/Batch Setter 提供只读数据，减少读取型 props 层层透传。
- Widget 元素注册配置 WidgetElementSchema 改为泛型接口 `WidgetElementSchema<TMetadata extends WidgetMetadata = WidgetMetadata>`，metadata 字段类型随之参数化；默认泛型保证读取处（WIDGET_ELEMENT_SCHEMAS、getWidgetElementSchema 返回值等）无需改动，各元素 schema 定义处可声明具体泛型获得写入端类型校验。textElementSchema 已改为 `WidgetElementSchema<WidgetTextElementMetadata>` 并移除冗余 satisfies。
- Widget 文本元素尺寸测量（measureWidgetTextElementSize / createWidgetTextRenderSize）联动最大行数配置：超出 maxLines 的行不再参与 content 尺寸计算，避免文本被 CSS 截断后元素 box 仍按全部行算高度导致下方留白。
- 修复 Widget 文件加载时内部模型归一化回写导致首次进入即提示未保存的问题。
- 修复 Widget 父组合缩放预览时深层子节点只跟随位置、不同步缩放尺寸的问题。
- 修复 Widget 设置面板在元素模型更新后仍编辑旧元素引用的问题。
- 修复 Widget 设置页打开组件时优先使用缓存数据，导致保存成功后再次打开仍显示旧数据的问题。
- 修复 Widget 侧栏图层拖拽到最后一个子节点后无法移出到父级同层的问题，并在同层没有可展开组合时隐藏折叠占位。
- 修复 Widget 侧栏中组外图层拖到组合最后一个子节点下方时，投放提示线错误退到父级层级的问题。
- 修复 Widget 侧栏中组外图层拖到组合第一个子节点上方时未能作为组合顶部子节点插入的问题。
- 修复 Widget 侧栏中拖拽投放线命中组合但普通列表顺序未变化时，移动事件被公共拖拽组件吞掉导致无法移入组合的问题。
- 修复 Widget 新建组合时标题使用随机元素 ID 的问题，改为按现有组合标题递增生成“组合1”等名称。
