# 2026-07-10

## Added
- Widget 编辑页加载完成并激活时动态提供 `get_widget` 与结构化 Patch `edit_widget` AI 工具，支持严格校验、当前页面上下文和元素树历史记录。

## Changed
- Widget 编辑器 AI 上下文由页面管理，并通过 BWidget 的 `replaceDocumentValue` 事务更新文档，其中元素树替换进入画布历史。
- ChatRuntime 将 renderer 工具执行器快照绑定到本轮 `runtimeId`，不传递 Widget 页面 ID，也不在 tool-request 到达时重新读取当前页面上下文。
- `edit_widget` 不再声明自动安全批准，自动安全模式下仍需用户确认，确认文案包含目标 Widget 文件信息。
- 新增 BWidget 元素动作 hook，将方法动作参数解析与运行态调用从 `useElementValue` 拆出，简化按钮元素点击执行代码。
- 调整 BWidget 按钮元素阻断状态，仅在点击处理内拦截动作，不再写入原生 `disabled` 属性。
- 优化 PageSetter「尺寸」设置项提示文案，去掉「留空按内容自适应」并改用正式表述，最终为「设置组件运行时的显示尺寸。空间足够时保持设定尺寸，空间不足时自动等比缩小」，同步更新对应单元测试断言。

## Fixed
- 修复 BChat 模型流在工具调用中途失败时同时显示工具卡片和错误卡片的问题，未完成工具会直接收尾为失败状态。
- 优化 BChat runtime 错误事件去重逻辑，最新 assistant 工具片段已展示同一失败时不再额外创建错误消息。
- 修复 Widget 页面切换后迟到的 `edit_widget` tool-request 可能解析到新页面工具上下文的问题。
