# 2026-07-10

## Changed
- 为文件图标解析新增多种特殊 Markdown 文件名专用规则，Skill 文件树同步使用完整文件名解析，支持 `skill.md`、AI 助手配置文件和许可证文档展示对应 vscode-icons 图标，并保留普通 Markdown 图标行为。
- 新增 `developing-widget` 技能草稿，提供 Widget 目录生成流程、格式与运行时参考、最小模板及无依赖校验脚本。
- 兼容带单一顶层目录的 zip 导入包，自动识别其中的 `SKILL.md` / `widget.json` 并剥离资源路径外层目录。
- 扩展小组件创建弹窗导入能力，支持直接拖拽或选择 `widget.json` / `.json` 文件创建小组件，并在 `widget` 通用标识建议下保留用户已有标识。
- 为 BSectionItem 标签区域增加右侧扩展槽，并将 Widget 循环列数自适应切换改为标签右侧复选框。
- 支持 BWidget 循环列数配置为自适应模式，运行态可按页面可用宽度自动计算每行列数。
- 新增 BWidget 元素动作 hook，将方法动作参数解析与运行态调用从 `useElementValue` 拆出，简化按钮元素点击执行代码。
- 调整 BWidget 按钮元素阻断状态，仅在点击处理内拦截动作，不再写入原生 `disabled` 属性。
- 拆分 Widget 页面脚本逻辑到 `src/views/widget/hooks`，将会话、选区、图层动作、画布拖拽和多选设置处理抽离为独立 hook。
- 统一 Widget 侧栏图层事件为数组载荷，单图层与多图层操作共用 `select-elements`、`copy-elements`、`delete-elements` 和 `move-elements`。
- 调整 BChat 工具确认授权策略，非危险确认默认展示“本会话允许”和“始终允许”，危险确认仍不允许记住授权。
- 将文件图标解析函数 `getFileIconByName` 重命名为 `resolveFileIcon`，同步更新 `BRecent/Icon.vue`、`SkillFileTree.vue`、相关测试与文档。
- 调整 ChatRuntime 文件读取确认规则，用户主目录下的 `.agents` 与 `.tibis` 只读访问仅在真实路径仍位于对应目录内时免确认。

## Fixed
- 修复 BChat 模型流在工具调用中途失败时同时显示工具卡片和错误卡片的问题，未完成工具会直接收尾为失败状态。
- 修复 Skill 创建预览中文件树虚拟路径全局排序导致目录层级展示错位的问题。
- 优化 BChat runtime 错误事件去重逻辑，最新 assistant 工具片段已展示同一失败时不再额外创建错误消息。
- 修复 `test/components/BText/method.component.test.ts` 中 `uses the parameter header action...` 用例失败：该用例未按交互约束先选中函数就点击「添加参数」按钮，组件 `Method.vue` 以 `v-if="editingAction.method"` 限制该按钮仅在已选函数时渲染。改为先选择 `submitOrder` 函数（补齐 2 个参数行）再点击「添加参数」验证新增一行（共 3 行）。
- 修复 BChat runtime 确认请求未携带记忆字段时，“本会话允许”不会写入授权记录，导致同一工具重复弹出确认的问题。
- 补充 BChat runtime 确认授权窄范围回归测试，确保只允许本会话授权时不会写入始终允许记录。
