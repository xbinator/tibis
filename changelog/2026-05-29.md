# 2026-05-29

## Changed
- BEditor Rich 大文档解析增加最近文档缓存，切回未变更的大文档时复用已解析 JSON，减少重复 Markdown 解析卡顿。

## Fixed
- 修复 Skill 工具概率性不发送的问题：`allBuiltinTools` 在 `<script setup>` 阶段一次性创建时 `skillStore.initialized` 为 `false`，导致 `createBuiltinTools` 中 skill 工具条件检查失败、工具被永久丢弃。改为在 `getActiveTools()` 中每次动态检查 `skillStore.initialized` 并补入 skill 工具。
- 修复 Shell 命令在 skill 目录执行时 cwd 越界被拒的问题：`getAdditionalShellWorkspaceRoots` 闭包捕获了工具创建时的 `enabledSkills` 静态快照（为空），改为每次调用时从 store 动态读取。
- 优化 Shell 命令确认体验：安全分析无 risk finding 时直接放行（如 `ls`、`echo` 等安全命令），有 finding 时才弹窗让用户确认。

## Performance
- 优化流式消息渲染性能：`BMessage` 中 Markdown 解析改为 `requestAnimationFrame` 帧级节流，流式期间每帧最多执行一次 `marked.parse()` + DOM 更新，解决流消息时每 token 触发全量 Markdown 重解析导致页面卡顿的问题。
- 优化流式期间 Token 用量估算：`useContextUsage` 新增内容级缓存，仅在消息数量变化或内容增长超 200 字符时立即重算，否则降到 300ms 间隔更新，避免每条 token 触发 `convert.toModelMessages()` + `JSON.stringify` 全文序列化。
- `ConversationView` MessageBubble 添加 `v-memo`，已完成消息（`finished=true`）跳过 VNode 重渲染，流式期间仅最后一条活跃消息参与 diff。
- `useChatStream` 文本/思考 token 追加改为 rAF 缓冲合并，将高频 `appendText`/`appendThinking` 调用从每 token 一次降为每帧一次，减少 Vue 响应式调度频率。
