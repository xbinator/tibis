# 2026-05-06

## Added
- 聊天上下文压缩功能完整实现，包括：
  - 压缩策略判断模块（`policy.ts`）：双阈值触发（消息轮数和上下文字符体积）
  - 消息分类模块（`planner.ts`）：将消息分为保留原文、文件语义和可摘要三类
  - 规则裁剪模块（`summarizer.ts`）：对可摘要消息进行规则化裁剪
  - AI 摘要生成模块（`summaryGenerator.ts`）：调用 AI 模型生成结构化摘要，支持降级策略
  - 上下文组装模块（`assembler.ts`）：将摘要和消息组装为最终上下文
  - 协调层模块（`coordinator.ts`）：串联各模块，管理会话级压缩状态和并发锁
  - 数据库表 `chat_session_summaries`：持久化存储会话摘要记录
  - 存储层实现（`src/shared/storage/chat-summaries/`）：支持 SQLite 和 localStorage 降级
  - 在 `ModelServiceType` 中添加 `'summarize'` 服务类型
- 添加摘要查看功能：
  - 创建 `SummaryModal` 组件显示摘要详细信息
  - 支持查看基本信息、摘要文本、结构化信息和元数据
  - 集成到聊天侧边栏，通过压缩按钮菜单打开
- 编写集成测试：
  - 创建 `compression.integration.test.ts` 测试文件
  - 测试 useCompression hook 的状态和方法
  - 测试 SummaryModal 组件的挂载
  - 测试压缩协调器的初始化和消息处理
  - 所有测试通过（8 个测试用例）
- Source 编辑器自定义 Selection 绘制扩展（`sourceEditorDrawSelection.ts`），用 ViewPlugin + Decoration.mark() 替代 CodeMirror 原生 selection 视觉渲染
- Suppression 机制（`suppressSourceEditorSelectionDraw` / `restoreSourceEditorSelectionDraw`），支持在保留真实选区的前提下临时隐藏普通 selection 装饰
- `EditorView.theme()` 统一覆盖 `.cm-selectionBackground` 和 `::selection` 为 transparent，让 decoration 成为唯一可见的 selection 表示
- 新增 `docs/superpowers/specs/2026-05-06-speech-runtime-v2-design.md`，定义语音运行时第二版方案，覆盖多官方模型并存、第三方模型路径注册、后台检查/下载更新、受控切换与回滚、以及从第一版目录结构迁移到 `state.json` 仓库结构的设计。
- 新增 `docs/superpowers/plans/2026-05-06-chat-file-reference-navigation.md`，将聊天文件引用导航拆分为共享解析、通用 `openFile(options)` 入口、editor 选区意图消费、source/rich 双模式选区实现和回归测试任务。
- 新增 `docs/superpowers/specs/2026-05-06-chat-context-compression-design.md`，定义 AI 聊天长会话的混合模式上下文压缩方案。

## Changed
- 集成上下文压缩功能到聊天流程（`useChatStream.ts`）：
  - 在发送新消息前自动检查是否需要压缩
  - 支持压缩失败时降级到原始消息
  - 清空模型消息缓存以适应压缩后的消息结构
- 修复聊天上下文压缩接入问题：
  - 修正发送链路中当前用户消息被重复拼接到模型上下文的问题
  - 修正压缩后 `preservedMessages` 与近期原文消息的组装边界，避免重复注入和遗漏最近消息
  - 为手动压缩增加独立入口，支持绕过自动阈值并以 `manual/full_rebuild` 语义生成摘要
  - 修正 `chat-summaries` 的 localStorage 降级路径返回最旧有效摘要的问题
  - 在模型服务设置页补充 `summarize` 服务配置入口
- 摘要模型调用改为接入 AI SDK 结构化输出：
  - 在 `AIRequestOptions` 中新增 `output` 配置与 `AIInvokeResult.output`
  - 主进程 `AIService.generateText()` 支持将 JSON Schema 转换为 `Output.object(...)`
  - `summaryGenerator.ts` 为会话摘要请求传入结构化 schema，并优先消费结构化对象结果
- 添加手动压缩功能：
  - 创建 `useCompression` hook 提供压缩管理能力
  - 创建 `CompressionButton` 组件提供 UI 入口
  - 在聊天侧边栏 header 添加压缩按钮
  - 支持查看压缩状态和当前摘要信息
- 统一聊天消息区与 Prompt 编辑器中文件引用 chip 的渲染结构，新增 `src/components/BChatSidebar/components/FileRefChip/` 承载共享展示数据与样式，并将 `BubblePartUserInput.vue`、`chipResolver.ts` 同步切换到共享实现
- `sourceSelectionAssistant.ts`：`clearNativeSelection()` 从空操作改为调用 `suppressSourceEditorSelectionDraw`，实现视觉层隐藏
- `sourceSelectionAssistant.ts`：`clearSelectionHighlight()` 在清除 AI 高亮后追加调用 `restoreSourceEditorSelectionDraw`，恢复普通 selection 绘制
- `PaneSourceEditor.vue`：在 extension 链中插入 `createSourceEditorDrawSelectionExtension()`
- `SelectionToolbarSource.vue`：工具栏定位改为基于 `getBoundingClientRect()` 的真实尺寸和容器边界动态收敛，并在首次测量完成前保持隐藏，修复选区靠右时浮层被压缩、靠顶部时浮层被遮挡以及首帧跳动延迟的问题
- `SelectionToolbarSource.vue`：source 编辑区收到 `pointerdown` 时立即隐藏 toolbar，避免上一轮工具栏在开始新一轮拖拽选区的瞬间继续可见
- `SelectionAIInput.vue`：AI 面板定位改为基于 `getBoundingClientRect()` 的真实尺寸与容器边界动态收敛，并在首次测量完成前保持隐藏，修复右侧溢出、底部空间不足时面板被遮挡以及首帧跳动延迟的问题
- `useSelectionAssistant.ts`：source 模式在编辑器内部开始新一轮 pointer 选区时立即清除上一轮持久高亮，修复第一段选区高亮要等第二次选区完成后才消失的问题
- `useSelectionAssistant.ts`：source 模式拖拽选区期间强制隐藏 `SelectionToolbar`，即使按下后立刻收到选区更新也不显示，仅在 `pointerup` 完成后按最终选区显示工具栏，修复鼠标按下时 toolbar 提前出现的问题
- `sourceSelectionAssistant.ts`：source 模式新增拖拽选区过程中的 `pointermove` 实时同步，修复选区高亮与工具状态只在 `mouseup` 后才更新的问题
- `PaneSourceEditor.vue`：移除 `.cm-selectionBackground` / `::selection` 的 CSS 规则（已由扩展内 theme 接管）
- `PaneSourceEditor.vue`：`source-ai-selection-highlight` 追加后代文字颜色覆盖，修复 `md-code-marker` 等 Markdown token 在 AI 高亮内未显示为白色的问题
- `PaneRichEditor.vue`：`ai-selection-highlight` 追加后代文字颜色覆盖，修复 `code`、链接等富文本 token 在 AI 高亮内未显示为选区前景色的问题
- `sourceEditorDrawSelection.ts`：`cm-custom-selection` 追加普通选区文字颜色与后代覆盖，修复 Source 普通 selection 中 Markdown / 代码 token 未显示为白色的问题
- `sourceEditorDrawSelection.ts`：移除“真实选区变化自动解除 suppression”的逻辑，改为仅在显式 `restoreSourceEditorSelectionDraw()` 时恢复普通 selection 绘制，避免 AI 面板打开期间普通 selection 重新露出
- 补充语音运行时后续演进方向，将现有“单运行时单模型”能力明确升级为“binary 仓库 + 模型仓库 + 当前选择指针”的设计思路。
- `docs/superpowers/specs/2026-05-05-chat-file-reference-navigation-design.md`：将文件引用导航方案调整为 `useNavigate.ts` 提供通用 `openFile(options)` 入口，并新增 `src/views/editor/hooks/useFileSelection.ts` 消费一次性文件选区意图的设计边界。
- `sourceLineMapping.ts`：rich 模式反向行号映射新增基于原始 Markdown 顶层 token 的对齐路径，修复空行分隔导致的文件引用跳转偏移问题。
- `docs/superpowers/specs/2026-05-06-chat-context-compression-design.md`：补充摘要覆盖边界、文件引用轻量保留、并发锁、摘要注入格式、状态失效规则与测试矩阵。
