# 2026-05-19

## Added
- 新增 macOS "文件右键 → 打开方式 → Tibis" 支持：在 `electron-builder.yml` 中注册 `.md/.markdown/.txt/.mdx/.json` 文件类型关联，主进程通过 `open-file` 事件接收文件路径并转发到渲染进程，由布局组件调用 `useOpenFile().openFileByPath()` 在编辑器中打开

## Changed
- 将 Vue Router 切换为 Hash History，避免 Electron `file://` 入口下应用路由被解析成本地文件路径
- 将 Vite 生产构建资源基础路径改为相对路径，修复 Electron 打包后通过 `file://` 打开应用时 `/assets/...` 被解析为磁盘根目录并 404 的问题
- 将前端构建期依赖（Vue/Monaco/Mermaid/TipTap/CodeMirror 等 58 个 Vite 可完整打包的包）从 `dependencies` 移至 `devDependencies`，仅保留主进程运行时必需的 12 个包，避免 electron-builder 将前端 node_modules 全部打包进最终安装包，预计缩小安装包体积 1GB 以上
- 移除 `electron-builder.yml` 中的手动 node_modules 白名单和 `"!node_modules/**/*"` 排除规则，改为依赖 electron-builder 自动处理 `dependencies` 及其传递依赖，修复因 pnpm 传递依赖未提升导致 `@ai-sdk/gateway` 等包在运行时找不到的问题
- 新增 `.npmrc` 配置 `public-hoist-pattern`，强制将 `@ai-sdk/*`、`@img/*`、`@opentelemetry/*` 传递依赖提升到顶层 `node_modules/`，确保 electron-builder 能在打包时解析到这些包
- 将 `@ai-sdk/gateway` 显式加入生产依赖，并补充 package 清单回归测试，确保 Electron 打包后的 AI SDK 运行时依赖完整
- 将 `zod` 显式加入生产依赖，修复 `@ai-sdk/provider-utils` 在打包后的 `app.asar` 中无法解析 `zod` 导致应用启动异常的问题
- 将 `@ai-sdk/provider`、`@standard-schema/spec` 和 `eventsource-parser` 显式加入生产依赖，修复 `@ai-sdk/provider-utils` 在打包后的 `app.asar` 中继续缺失流式解析相关运行时依赖的问题
- 将 `@ai-sdk/provider-utils` 和 `@opentelemetry/api` 显式加入生产依赖，补齐 `ai` 包在打包后直接导入的运行时依赖
- 将 `sharp` 加入 `pnpm.onlyBuiltDependencies`，确保其原生二进制在安装时正确编译
- 新增 `.github/workflows/release.yml`，支持推送 `v*` 标签后自动构建 macOS、Windows、Linux 安装包并发布到 GitHub Release
- 扩展 `.github/workflows/release.yml` 的手动触发流程，支持在 GitHub Actions 中选择版本递增类型，自动更新 `package.json`、提交发版 commit、创建标签并继续发布 Release
- 新增 `docs/development/github-release.md`，记录 GitHub Actions 自动发版、版本递增、标签触发和未签名安装包说明
- 更新 `README.md`，补充 GitHub Releases 下载发布版、平台产物和未签名安装包提示
- 修正 `electron-builder.yml` 的 Windows 平台配置键为 `win`，并在 `package.json` 中显式关闭 electron-builder 的隐式发布，避免 GitHub Release 打包阶段配置校验失败或重复发布
- 为 `test/package-scripts.test.ts` 补充 `electron:build` 脚本断言，防止后续移除 `--publish never` 导致 tag 打包时重复发布
- 为 `electron-builder.yml` 补充 Linux `category` 和 `maintainer`，满足 deb 安装包构建所需的维护者邮箱要求
- 在 `.github/workflows/release.yml` 的打包步骤显式设置 `ELECTRON_BUILDER_BINARIES_MIRROR`，避免 macOS DMG 构建从镜像下载 `dmg-builder` 时遇到 404
- 为 `.github/workflows/release.yml` 的手动发版流程补充取消回滚，用户取消后会删除已推送的版本 tag，并将分支回退到版本号提交前的提交
- 调整 `src/components/BJsonViewer/hooks/useJsonGraph.ts` 的树布局算法，改为按子树真实边界分配纵向空间，修复 `cs.json` 场景下部分节点纵向贴近或重叠的问题
- 调整 `src/components/BJsonViewer/hooks/useJsonGraph.ts` 的纵向居中逻辑，改为基于整张图的实际上下边界计算中心点，修复节点图整体未真正垂直居中的问题
- 参考 `AykutSarac/jsoncrack.com` 的父子层居中思路，重写 `src/components/BJsonViewer/hooks/useJsonGraph.ts` 的纵向排布策略，改为按子树总高度居中放置父节点，修复根节点与一级子节点从顶部对齐的问题
- 调整 `src/components/BJsonViewer/index.vue` 的首屏视口定位策略，改为先 `fitView` 再将视口中心对准根节点中心，修复首屏展示仍然偏上的问题
- 调整 `src/components/BJsonViewer/index.vue` 的首屏视口定位策略，移除额外的根节点聚焦，改回按整张图执行 `fitView`，修复整体视图未居中而是根节点居中的问题
- 调整 `src/components/BJsonViewer/index.vue` 的初始化展示策略，改为直接启用 `VueFlow` 内建 `fit-view-on-init` 做整图居中，避免外层手动时机导致整体视图偏移
- 调整 `src/components/BJsonViewer/hooks/useJsonGraph.ts` 的根数组建图逻辑，为数组根节点生成 `"[n items]"` 摘要节点并连接到各元素，修复 `cs.json` 为数组时未按参考图展示的问题
- 收紧 `test/components/BJsonViewer/useJsonGraph.test.ts` 的布局断言，补充子树间距与实际边界居中校验，防止后续回归
- 为 `test/components/BJsonViewer/useJsonGraph.test.ts` 补充“根节点中心与一级子节点整体中心对齐”断言，防止父子层级重新退回顶部对齐
- 为 `test/components/BJsonViewer/useJsonGraph.test.ts` 补充根数组摘要节点与嵌套数组直连行为断言，防止数组展示回归
- 新增 `test/components/BJsonViewer/index.test.ts`，校验 JSON 节点图初始化时会保留适配缩放并聚焦到根节点中心
- 更新 `test/components/BJsonViewer/index.test.ts`，校验 JSON 节点图初始化时仅执行整图 `fitView`，不再额外强制聚焦根节点
- 重写 `test/components/BJsonViewer/index.test.ts`，校验组件初始化时启用 `VueFlow` 内建 `fitViewOnInit`，确保整体视图居中行为稳定
- 移除 `electron/main/modules/store/service.mts` 对 Electron `safeStorage` 和 store 加密密钥文件的依赖，改为直接初始化普通 `electron-store`
