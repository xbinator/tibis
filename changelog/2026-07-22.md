# 2026-07-22

## Changed

- 修复 `BMessage` 在中文正文中渲染 `**无法真正"感受"**情绪`、`*无法真正。*体验`、`__无法真正"感受"__情绪` 等紧邻标点与 CJK 字符的强调内容时保留 Markdown 标记的问题。
- 将聊天会话集合、分页状态和加载状态统一收敛到 `useChatSessionStore`，聊天页创建、分支、改名或删除会话后会实时同步侧栏历史列表，并移除 `SessionHistory` 的局部数据加载与 `refreshSessions` 暴露接口；聊天页初始化改为静默加载，分页保护仅由 Store 负责，侧栏当前会话由 `useChatSession` 统一派生。
- 将顶部标签状态改为瞬时 `Tab.status` 写穿模型：状态来源在变化时同步通用 `TabStatus`，`HeaderTabs` 直接渲染标签状态且不再依赖聊天 Runtime Store；持久化快照会排除运行状态，并由关闭守卫统一清理已关闭聊天的 Runtime。
- 修复聊天页本地消息回退会短暂刷新顶部标签加载状态的问题，回退期间仍锁定聊天交互但不再发布 Runtime running 状态。
- 修复打开持久化聊天页时顶部标签仍显示默认“聊天”的问题，聊天页宿主会将标签标题同步为当前会话标题。
- 修正通用标签状态边界：新增和替换标签会保留调用方显式状态，Tabs Store 实例不再共享默认集合；聊天 Runtime 记录初始化不再隐式修改标签状态，改由聊天页显式恢复已有状态。
- 将 `BChat` 的运行状态通知合并为单个 `runtime-status-change` 事件，成功完成通过 `{ status: 'completed', sessionId }` payload 表达，页面宿主不再监听独立完成事件。
- 内联聊天页完成态处理逻辑，减少运行状态事件分流时的重复包装函数。
- 简化聊天页活动标签路径判断，拆开已有标签路径和兜底路径计算。
- 将聊天页当前归属标签激活态抽为 `ownerActive` 计算属性，保留 `isTabActive(tabId)` 处理指定标签判断。
- 将聊天标签 Runtime Store 从 `src/stores/chat/tabRuntime.ts` / `useChatTabRuntimeStore` 重命名为 `src/stores/chat/tab.ts` / `useChatTabStore`，并同步更新源码与测试导入路径。
- 将聊天工具权限 Store 从 `src/stores/chat/toolPermission.ts` / `useToolPermissionStore` 重命名为 `src/stores/chat/permission.ts` / `useChatPermissionStore`，并同步更新源码与测试导入路径。
- 将 `useChatOwner` 调整为 `useChatRoute`，由 hook 统一处理 ChatSider 的已打开聊天页复用、打开聊天页和删除会话后的标签同步。
- 调整 `BChat` 模型切换边界：新会话草稿继续更新全局默认模型，已有会话改为按会话维护内存临时模型，工具栏与 `/model` 命令面板复用同一切换入口，并将模型快照随发送、重新生成、用户选择续跑及上下文压缩请求传给主进程，避免不同会话互相覆盖默认模型。
- 将 `BChat` 已有会话的模型选择持久化到 `chat_sessions.metadata_json`：新会话在首次发送时记录模型，旧会话在首次 Runtime 前按需补写，分支继承源会话模型；模型切换不再影响全局默认，并在发送、重新生成、用户选择续跑和上下文压缩前保证会话模型已落库。
- 修复 `BChat` 首次挂载时 Runtime 模型解析早于默认模型和 Provider 初始化的问题，并合并并发 Provider 读取；新增 Electron Node 模式数据库测试命令，使 `metadata_json` 增量迁移在标准测试链路中真实执行。
- 收敛 ChatRuntime 请求配置类型，统一复用必填模型快照，并移除 Renderer 请求准备阶段的重复模型覆盖。
- 收敛模型选择器的 Provider 初始化归属，由 Chat Composer 统一加载模型来源，展示组件不再重复读取 Provider 配置。
- 将 `BSmartEditor` 的 CodeMirror 内容滚动交给 `BScrollbar` 承载，统一提示词编辑器滚动条显示。
