# 2026-06-03

## Changed
- 重构最近记录系统为混合类型架构，支持文件与 WebView 网页统一管理。StoredFile 新增 `type` 字段，新增 WebviewRecord、RecentRecord 类型，新增 useRecentStore，原 files store 改为兼容层委托。
- BSearchRecent 搜索弹窗、欢迎页、WebView 页面适配新的混合记录类型，网页打开后自动记录到最近列表。
- 各模块（编辑器自动保存、文件状态、会话、FileReadTool、BChatSidebar、导航 hooks）适配 StoredFile 新增 `type` 字段，useNavigate 重命名 `openInWebview` → `openWebview`。
- 修复 WebView 页面元素选择器在页面滚动、刷新后的状态同步问题，并在选中元素后保留页面内选中描边。
- 补充当前网页读取工具在聊天侧边栏中的展示标签与结果摘要。
- 调整 WebView 地址栏操作区，在保留"在浏览器打开"按钮的同时新增更多操作菜单。
- 修复 WebView 完整页面长截屏时定位层重复拼接的问题，并按首屏吸顶、末屏吸底规则过滤中途出现的 `fixed` / `sticky` 元素。
- 将 WebView 页面中的截图与缓存操作分别抽离到独立 hooks，收敛 `index.vue` 的页面工具逻辑。
- 修复 AI 聊天首次使用 MCP 时未自动启动 server 的问题，聊天请求会在注册 MCP 工具前补齐 discovery 缓存。
- 优化聊天工具调用展示，将 MCP SDK 编码工具名显示为可读的原始 MCP tool 名称。
- 修复 MCP 设置页首次进入时不自动发现工具的问题，并在重启后直接使用返回的 discovery cache 展示工具列表。
- 修复 MCP 等流式回复完成后 assistant 消息 `content` 未同步追加文本，导致最终回复内容字段未落库的问题。
- 将 MCP server 配置迁移到 `~/.tibis/settings.json` 的 `mcp.servers` 字段，保留 OAuth 凭据在 Electron Store、discovery cache 在主进程内存。
- 修复 provider 列表读取时未按 `settings.json` 数组顺序输出的问题，未记录的内置 provider 继续按默认顺序追加。
- 收敛工具设置保存异常处理，业务页面移除空 catch，底层通过 `asyncTo` 兼容 `settings.json` 读写失败。
- 记忆功能 `memory_enabled` 开关从 localStorage 迁移到 setting store（`memoryEnabled`），业务层直接使用 useSettingStore 读写，移除 memory store 中间层 `enabled`/`setEnabled`。

## Features
- BSearchRecent 搜索弹窗支持输入 http/https URL，在应用内 Webview 中打开链接。
- 新增当前网页读取 AI 工具，支持在激活 WebView 时按需读取页面快照。
- WebView 页面新增当前视图截屏、完整页面长截屏和清除缓存能力，截屏直接弹出 PNG 保存对话框。
