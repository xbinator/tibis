# 2026-07-06

## Changed
- [BWidget/widgetLoop] `createElementLoopRenderElements` 在判断是否走循环展开逻辑时，新增对 `loopConfig.source` 的校验：仅当 `enabled` 为 `true` 且 `source` 非空时才进入 `expandLoopTemplateTarget`，否则回退到普通元素渲染（元素 + 子元素渲染一次）。
- [BWidget/widgetLoop] `createElementLoopRenderElements` 内局部变量 `config` 重命名为 `loopConfig`，明确其为元素循环配置的语义；`WidgetLoopTemplateTarget.config` 属性名保持不变。

## Features
- [BWidget/widgetLoop] 修复循环启用但未配置数据源（`source: ''`）时元素整体消失的问题：现在会回退到普通渲染，保证元素至少渲染一次，避免配置不完整导致 UI 空白。

## TDD 流程记录
本次改动严格遵循 TDD（Test-Driven Development）流程：

1. **RED - 写失败测试**：在 `test/components/BWidget/widget-loop.test.ts` 新增测试 `falls back to normal rendering when loop is enabled but source is empty`，验证 `enabled: true` + `source: ''` 时元素应渲染一次（期望 `['text-1']`）。
2. **验证 RED**：运行测试确认失败，失败原因为 `expected [] to deeply equal [ 'text-1' ]`，即空 source 导致 `expandLoopTemplateTarget` 返回空数组，元素消失。
3. **GREEN - 最小实现**：在 `src/components/BWidget/utils/widgetLoop.ts` 的循环判断条件中加入 `&& loopConfig.source`，让空 source 时跳过展开逻辑，回退到普通渲染路径。
4. **验证 GREEN**：运行测试确认 11 个测试全部通过（新增 1 个 + 原有 10 个无回归）。
5. **REFACTOR**：代码已是最简形式，无需重构。

## Features
- [BText/Input] `BTextInput` 新增 `useTemplateSyntax` prop（默认 `true`，保持向后兼容）：控制变量下拉选中后是否用 `{{ }}` 模板语法包裹变量值。
  - `true`（默认）：插入 `{{ $input.items }}`，适用于模板字段（如图片地址 `element.metadata.src`）。
  - `false`：插入纯路径 `$input.items`，适用于绑定路径字段（如循环数据源 `element.loop.source`）。
  - `handleVariableSelect` 根据 `props.useTemplateSyntax` 选择插入格式，消费方无需自行去包裹。

## Fixed
- [BWidget/widgetDataSchema] 修复 `inferSchemaPropertyFromExpression` 兜底返回 `{ type: 'string' }` 污染已有类型的 bug：当 Widget 脚本中 `data` 字段用空数组初始化（如 `movieList: []`），随后在 `mounted/methods` 中用无法静态推导的表达式赋值（如 `this.movieList = res.data.movieList`，`res` 来自异步请求）时，`movieList` 的 schema 会被错误覆盖为 `{ type: 'string' }`，导致循环数据源 schema 解析失败、迭代变量不出现。
  - `inferSchemaPropertyFromExpression` 无法识别的表达式现在返回 `undefined` 而非 `{ type: 'string' }`。
  - `mergeSchemaProperty` 在 `next` 为 `undefined` 时保留 `current`，避免无法推导的赋值覆盖已有类型。
  - `writeDataSchemaProperty` 在 `property` 为 `undefined` 时跳过写入。
  - `ObjectLiteralExpression`/`ArrayLiteralExpression` 分支同步适配 `undefined` 返回值。

## TDD 流程记录

4. **验证 GREEN**：新测试通过，widget-data-schema（11）+ widget-loop（11）+ use-element-variables（14）共 36 个测试全部通过，无回归。tsc + eslint 通过。
5. **REFACTOR**：代码已最简，无需重构。

## Changed
