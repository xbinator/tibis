# 2026-05-18

## Changed
- 打开已不存在的文件时弹窗提示并自动从最近文件列表中移除，由 src/hooks/useOpenFile.ts 中 openFileByPath 统一处理
- BEditor 的快捷操作新增“导出 PDF”，并在 Markdown.vue 内按当前富文本/源码模式直接组装导出 HTML 后交给原生层落盘
- 修复 BEditor 富文本 PDF 导出保留编辑器视口宽度与横向外边距导致内容偏右的问题
- 修复 BEditor 源码模式 PDF 导出会注入可见 div 包装并带灰色代码块背景的问题
- MCP Server 编辑弹窗由内置 CodeMirror JSON 编辑器切换为 BMonaco，并同步更新相关测试桩与校验流程
- 为 BMonaco 补充宿主与降级态最小高度样式，修复编辑器区域未撑开导致无高度的问题
- FrontMatterCard 组件 formatComplexValue 新增 Date 类型处理，修复 Date 值显示为 `{0 个属性}` 的问题
- FrontMatterCard 组件复杂值编辑器由内联 Transition 弹层重构为 BModal + ATextarea，优化交互体验
- 重构 stores 目录结构，按业务领域（editor/chat/ai/ui/workspace/helpers）分组
- 提取持久化中间层 helpers/persist.ts，统一 loadPersistedState/persistState 模式
- 提取事件总线 helpers/events.ts，解耦 store 间直接依赖
- 从 setting store 拆分 toolPermission store 到 chat/toolPermission.ts
- editorFileWatch 与 tabs 解耦，通过事件总线通信
- setting store 和 tabs store 重构为使用 persist 中间层
- useChatStore 重命名为 useChatSessionStore
- 新增 BEditor 统一化设计与实现计划文档，明确取消 BMarkdown 组织并收口到 BEditor
- 将原 BMarkdown 的 components、hooks、adapters、extensions、utils 整体迁入 BEditor 域
- 将 editor 页面入口改为固定渲染 BEditor，并移除基于 driver 的编辑器分发
- 将编辑器工具上下文创建统一收口到 BEditor
- 将 Monaco 低层实现从 BEditor 内部 pane 抽离为独立 BMonaco 组件
- 将 BEditor 内部目录继续工程化，拆分为 panes/shared，并清理空 implementations 目录
- 将选区工具栏和 AI 输入的宿主层从 Markdown pane 内上提到 BEditor
- 清理 BEditor 域内残留的 BMarkdown 类型别名，统一为 Editor* 命名
- 将测试目录与测试文案中的 BMarkdown 语义迁移到 BEditor
- 将 BEditor index.vue 中的 Markdown 布局分支抽离为独立组件 Markdown.vue，精简 index.vue 职责为编辑器类型路由与统一调度
- 修复 BEditor 富文本代码块无语言时首次进入会被自动识别并错误上色的问题，并补充 Vue/React 语言别名
