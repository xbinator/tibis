# 2026-05-31

## Added
- 新增多主题预设系统：引入 BasePalette 接口和 createThemeTokens 工厂函数，每个主题只需声明约 30 个基础色，由工厂函数集中派生完整的 ThemeTokens（约 180 个字段）
- 新增主题注册表 registry.ts，提供 registerPreset、getPresetList、getResolvedTokens API，支持主题的注册、查询和 fallback
- 新增 10 个主题预设（每个均提供 light 和 dark 两套色值）：
  - Ayu：基于 dempfi/ayu 主题
  - Catppuccin：基于 catppuccin/catppuccin 主题（暗色使用 Mocha 变体）
  - Catppuccin Macchiato：基于 catppuccin Macchiato 变体
  - Everforest：基于 sainnhe/everforest 主题
  - Gruvbox：基于 morhetz/gruvbox 主题
  - Kanagawa：基于 rebelot/kanagawa.nvim 主题
  - Matrix：黑客风格黑底绿字主题
  - Nord：基于 arcticicestudio/nord 主题
  - One Dark：基于 Atom One Dark 主题
  - Tokyonight：基于 folke/tokyonight.nvim 主题
- 设置页面新增「主题风格」选择器，支持在已注册的主题预设间切换
- 视图菜单新增「配色方案」子菜单，包含明暗模式和主题风格选项
- 首屏防御脚本支持读取 themePreset，减少非默认预设首次加载闪烁
- AI 设置工具新增 themePreset 设置键，支持大模型读取和切换主题色预设

## Changed
- 重组 src/theme 目录结构为分层式：types/（纯类型）、core/（基础设施：factory/registry/derive/apply）、presets/（纯数据）、index.ts（统一入口）
- 删除 tokens.ts（8行重导出）和 tokens-interface.ts（连字符命名），合并为 types/tokens.ts
- 将 factory.ts 从 presets/ 移至 core/（工厂函数不是预设数据）
- 将 registry.ts、derive.ts、apply.ts 从根目录移至 core/
- 将原 tokens.ts 中的 light/dark 主题迁移至 presets/default.ts，作为默认主题预设注册
- useAntdTheme 改为从注册表获取当前预设 Token 派生 Ant Design 主题配置
- Monaco 编辑器主题名改为 tibis-{presetId}-{mode}，支持基于预设的动态注册
- setting store 新增 themePreset 字段和 setThemePreset action

## Removed
- 删除记忆自动提取管道：移除 `src/ai/memory/extractor.ts`、`src/ai/memory/merger.ts`，不再通过独立 AI 调用提取对话记忆
- 删除 `ExtractedMemory`、`ExtractedMemoryItem`、`ExtractionMessage` 类型定义
- 删除 MemoryStore 中的 `extractFromConversation`、`onTurnComplete`、`getChatModelConfig` 方法
- 删除 `useChatStream.ts` 中的 `triggerMemoryExtraction` 函数和调用点

## Changed
- 记忆管理改为对话 AI 直接操作模式：新增 `edit_memory` 内置工具（`src/ai/tools/builtin/MemoryTool/`），支持 read/add/update/remove 四种操作
- 记忆 Store 精简为纯缓存层：保留 `loadMemory`、`saveMemory`、`buildSystemPromptContext`、`updateItem`、`deleteItem`、`clearAll`、`setEnabled`
- `edit_memory` 工具注册为内置 write 工具，`safeAutoApprove: true`，无需用户确认
- `edit_memory` 工具改用 Store API 操作而非直接文件 IO，消除数据源分叉
- `resolveTargetIndex` 移除包含匹配，仅保留精确匹配 + 显式 index
- 添加单分区条目上限（20 条），防止 AI 滥用导致记忆膨胀
- 优化工具 description 引导 AI 主动发现并保存偏好/事实

## Added
- 新增 `test/ai/tools/builtin-memory.test.ts`（20 个测试），覆盖 read/add/update/remove 全操作、边界情况、Store 一致性
