# 2026-07-13

## Added

- Widget 与 Skill 设置列表新增设置菜单，支持打开对应编辑器并将整个资源目录移入系统回收站。

## Changed

- BDropdownMenu 默认菜单项支持按 `DropdownOption.icon` 与 `iconSize` 渲染图标，确保设置菜单等调用方的操作图标正常展示。
- 命令式删除确认弹窗通过关闭按钮、遮罩或 Esc 退出时统一返回取消结果，避免调用方删除流程持续锁定。
- 修复 BWidget 运行态配置展示宽高后只能随宿主缩小、无法恢复设定尺寸的问题；运行态内容现在会在空间不足时等比缩小，并在空间恢复后放大回设定尺寸。
- 简化 BWidget 运行态布局 Hook，将基础盒子、缩放和偏移的过度拆分逻辑合并为线性布局计算，保持现有展示尺寸行为不变。
- Widget 组件侧栏按“基础”和“交互”分类展示元素，元素分类由各自 Schema 声明，方便后续扩展。
- QuestionCard 等待用户选择期间保持 assistant 消息与 Turn 为 loading/waiting 状态，历史和实时消息统一修正矛盾状态，仅在用户主动终止或后续生成完成时进入终态。
- ChatRuntime 完成事件增加明确的等待原因，Question 消息仅在旧 Runtime 释放会话写锁后发布；中止或失败时同步将待回答工具结果收敛为 cancelled/failure，避免残留待回答状态。
- 待回答 Question 统一建模为可从持久化消息恢复的 PendingInteraction，刷新后同步恢复 Session、Turn 与 Agent 等待状态；答案提交增加 single-flight 与组件锁定，防止重复点击启动多个 Runtime。
- Widget 选区 hook（`useSelection`）处理器重命名：移除 `Widget` 前缀并区分语义，`handleWidgetValueUpdate`→`handleDataUpdate`、`handleWidgetTargetUpdate`→`handleSelectUpdate`、`handleWidgetSelectionUpdate`→`handleSelectionUpdate`，同步更新私有函数 `shouldIgnoreWidgetTargetUpdate`→`shouldIgnoreSelectUpdate` 及 `src/views/widget/index.vue` 的模板事件绑定与解构。
- Widget 选区 hook（`useSelection`）状态重命名：`activeSidebarElementId`→`activeElementId`，同步更新 `src/views/widget/index.vue` 的模板绑定值与 `useSelection` 解构（prop 名 `active-element-id` 保持不变）。
- WebView 标签页标题同步 hook 重命名：`src/views/webview/shared/hooks/useWebviewTabTitle.ts`→`useTabTitle.ts`，导出函数 `useWebviewTabTitle`→`useTabTitle`，同步更新 `src/views/webview/web/index.vue`、`src/views/webview/native/index.vue` 及 `test/views/webview/` 下相关测试的模块 mock 与导入路径。
- Widget 图层操作 hook（`useLayerActions`）重构为命名空间 API：返回值类型 `UseLayerActionsReturn`→`LayerActions`（方法 `select`/`copy`/`remove`/`move`），内部单数委托函数去掉 `handleSidebar` 前缀；`move` 参数对象化为 `LayerMovePayload`（`sourceIds`/`targetIds`/`position`），同步更新 `src/views/widget/index.vue` 调用点（`const layer = useLayerActions(...)` 与模板 `layer.xxx` 绑定）、`PanelSidebar.vue` 与 `SidebarLayer.vue` 的 `move-elements` 事件协议（emits 类型与 emit 调用改为 payload 对象），以及 `test/views/widget/index.test.ts`、`test/views/widget/sidebar-layer-selection.test.ts` 的 emit 断言。
- 化简 `src/views/widget/utils/layerOrder.ts` 长函数：抽取私有 helper `reorderSiblingsWithinParent`（统一两个 `reorder` 公开函数的同级内 before/after 移动逻辑）与 `placeMovedElementInTarget`（统一 `moveWidgetLayerElementAcrossParents` 的 inside / before-after 对称分支），三个 ~44 行的长函数被化简为更短的函数并复用上述 helper，消除了重复的 reverse-操作-reverse 与对称分支逻辑；公开 API、调用方与测试均保持不变（因新增 helper 的 JSDoc 与入参校验，文件总行数由 294 微增至 317）。
- 重构 BWidget Board 命令组织，将创建、剪贴板和状态命令改为单文件具名函数结构，函数型公开成员统一使用 `on` 前缀，并移除未使用的 `updateElementStyle`、`reorderElement` API。
- 修复 Widget 属性、批量设置和图层操作更新后无法撤销与重做的问题，并确保运行代码或 Schema 等同文档配置更新不会误清历史。

## Features

- AGENTS.md 新增函数命名规范：函数名不超过 4 个单词。
- Widget 选区 hook（`useSelection`）返回值改为扁平结构：`state`/`handlers` 命名空间合并为 `{ selectedTarget, selectedElementIds, activeElementId, onDataUpdate, onSelectUpdate, onSelectionUpdate }`，调用方 `index.vue` 改为直接解构。
- Widget 多选 hook（`useMultiSelection`）返回值改为扁平结构：`handlers` 命名空间合并为 `{ onMultiCommand, onElementCommand, onMultiLayoutChange, onMultiStyleChange }`。
- `src/components/BWidget/utils/widgetTree.ts` 函数名精简至 ≤4 词：`findWidgetElementTreeNode`→`findElementTreeNode`、`updateWidgetElementInTree`→`updateElementInTree`、`removeWidgetElementFromTree`→`removeElementFromTree`、`getWidgetElementAbsolutePosition`→`getAbsolutePosition`、`getWidgetElementParentLocalPosition`→`getLocalPosition`、`replaceWidgetElementSiblingList`→`replaceSiblingList`、`isSameWidgetElementParent`→`isSameParent`，同步更新全部 12 个源文件和 2 个测试文件的引用。

- 移除多余的二次包装函数：`src/components/BWidget/utils/widgetGeometry.ts` 中的 `findWidgetShapeElement` 和 `src/components/BWidget/index.vue` 中的 `getShapeElementById`，调用点直接内联 `findElementTreeNode`。
