# 2026-06-05

## Changed
- 调整模型服务商列表合并顺序，内置服务商始终按默认配置顺序展示，自定义服务商追加在末尾。
- 改进聊天上下文压缩摘要保真度，避免用户提供的代码清单、排序规则和输出要求被压缩为单一句话题。
- 将聊天上下文压缩边界渲染为 Markdown 交接稿，并对齐多段摘要召回的渲染格式。
- 引入通用长聊天压缩摘要 v3 兼容视图，新压缩记录写入 `generalSummary` 并保留旧摘要字段。
- 修正多段摘要召回预算估算，避免 Markdown 交接稿比旧摘要文本更长时超量注入历史段。
- 将聊天上下文压缩 tail 原文保留升级为按模型上下文窗口预算扩展，同时至少保留最近两个用户轮次。
- 为聊天上下文压缩增加开发者可观察性日志，记录压缩比例、原始需求数量和未完成事项数量。
- 修正 WebView 模块的 Electron 类型引用方式，使渲染端 TypeScript 全量检查可通过。
- 补充手动上下文压缩 hook 集成测试，覆盖 tail 保留、Markdown 边界回填和压缩指标日志。
- 修正聊天输入栏上下文用量口径，改为按当前模型消息切片估算，并区分会话累计 token 消耗。
- 修复聊天流式完成时最后一帧文本未先刷新就落库的问题，避免切换会话后刚完成的 AI 回复丢失。
- 修复 MCP/Skill 工具续轮时中间工具结果落库与最终回答落库乱序的问题，避免最终内容被旧快照覆盖。
- 调整聊天气泡加载态传递规则，等待用户回答问题时不再显示额外 loading，其余加载场景保持显示。
- 为记忆整理增加普通文本 JSON fallback，兼容不支持结构化输出的聊天模型。

## Features
- 为 BMessage Markdown 图片接入 BImageViewer 预览，点击图片可放大查看并避免与文本选区冲突。
