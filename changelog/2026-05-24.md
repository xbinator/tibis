# 2026-05-24

## Added
- Skill 系统：支持通过 SKILL.md 文件定义可复用的领域专用工作流指令，LLM 自动判断何时加载
- Skill 解析器：解析 SKILL.md frontmatter（name/description）和 body 指令内容
- Skill 扫描器：自动扫描项目 .agents/skills/ 和用户 ~/.agents/skills/ 目录
- Skill 工具：注册 skill 工具，LLM 通过工具调用按需加载 Skill 指令
- Skill Pinia Store：管理 skill 列表、启用状态、扫描配置和持久化
- FileWatchService 目录监听：扩展支持目录级监听，实时响应 skill 文件变化
- 设置页 Skill 管理 UI：统一目录说明、Skill 列表展示、启用/禁用开关
- 聊天侧边栏 Skill 提示：LLM 调用 skill 工具后显示标签提示
- Skill 只读详情抽屉：支持点击 Skill 列表项浏览 Skill 目录文件树并预览文件内容

## Changed
- MCP 本地 stdio runtime 改为可复用会话生命周期，支持主进程连接、断开与重启并复用已连接 server 执行工具。
- BPanelSplitter 的 minWidth/maxWidth 支持百分数字符串，并通过 VueUse 在父容器尺寸变化时重新换算拖拽约束。
- AIToolDefinition.description 支持函数类型，实现 skill 工具动态描述生成
- SkillScannerAPI 接口适配 native.readWorkspaceDirectory 类型（directoryPath + type 字段）
- SkillStore 新增 rescan 方法，设置页刷新按钮触发真正重新扫描
- BChatSidebar 集成 skillStore 初始化、目录监听和增量更新
- createBuiltinTools 仅在 skillStore 初始化完成后注册 skill 工具，description 动态反映可用 skill
- SKILL_TOOL_NAME 加入默认只读工具列表
- Skill 扫描路径统一为当前工作区 `.agents/skills/<name>/SKILL.md`，移除自定义搜索路径
- Skill 目录监听默认只广播 SKILL.md 变化，避免资源 Markdown 文件触发错误解析
- Skill 工具动态 description 增加长度上限，避免大量 skill 挤占上下文
- Skill 设置页右侧详情面板支持关闭，点击列表项可重新打开
- 设置页侧边栏折叠态新增菜单项 Tooltip，并为折叠按钮补充状态提示

## Fixed
- 修复 CodeBlock Mermaid 首次渲染显示异常的问题：BSuspense 的 v-if 导致 mermaidPreviewRef 延迟挂载，单次 nextTick 无法保证 DOM 就绪；添加 waitForMermaidPreviewRef 递归等待函数（nextTick + requestAnimationFrame 轮询），确保 ref 挂载后再执行渲染
- 修复多个 Mermaid 代码块首次渲染时可能复用同一 Mermaid DOM id，导致首屏图形渲染异常的问题
- 修复 Skill 初始化清理函数在热更新/卸载时可能引用已失效 `offSkillChanged` 标识符的问题，统一改为 cleanup 回调列表并清理目录监听
