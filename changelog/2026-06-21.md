# 2026-06-21

## Added
- BEditor Markdown 编辑器新增基于适配器的 AI 内联补全能力，支持 Rich 与 Source 两种编辑模式共用状态机。
- 新增火山引擎独立请求格式，支持把通用 reasoning 开关映射为火山方舟 `thinking.type` 参数。
- 新增 Moonshot、GLM、MiniMax、MiMo 独立请求格式，并支持 Kimi 思考模式的安全启用映射。

## Changed
- 修复聊天历史分页在同一创建时间下重复加载已展示消息的问题。
- 修复 Rich 编辑器 inline code 内 AI 选区高亮无法覆盖 code 左右 padding 的问题，并避免中间选区误补偿。
- 修复纯文本 AI 调用在未请求结构化输出时误读 SDK output 导致 NoOutputGeneratedError 的问题。
- 移除 BEditor 内联补全的显式输出 token 限制，避免为补全请求固定 max_tokens。
- 将 BEditor 内联补全的通用 reasoning 禁用语义交由 DeepSeek provider 映射为 thinking disabled，避免 OpenAI 兼容接口收到无效 reasoning_effort。
- 将 BEditor pane 级内联补全编排迁移到 useInlineCompletion，复用 Rich 与 Source 的 AI 调用和状态机生命周期逻辑。
- 收紧 BEditor 内联补全 prompt 与输出裁剪，避免模型返回整段 Markdown 内容污染 ghost text。
- 修复 BEditor 内联补全 ghost text 可见时 Tab 被编辑器默认按键逻辑消费，导致无法接受补全的问题。
- 修复 Rich 内联补全接受时插入换行，并避免接受补全产生的 input 事件再次触发模型请求。
- 修复 Rich 接受内联补全后与上一段用户输入合并到同一个 undo 历史事件，导致无法继续回退更早编辑的问题。
- 修复 Rich undo/redo 历史事务被内联补全误判为用户输入，从而再次触发模型请求的问题。
- 修复 Rich 删除类编辑被内联补全误判为用户输入，从而触发模型请求的问题。
- 收敛 BEditor 内联补全触发边界，仅在用户正向文本输入导致文档增长时触发，避免格式变化、等长替换、Source 删除/撤销和空输入法组合结束误触发模型。
- 修复 Source 内联补全在用户输入后的 keyup 事件中被误判为光标移动，导致补全请求 debounce 被取消而无法触发模型的问题。
- 将 BEditor 内联补全 stale 校验改为 Rich/Source 单调文档修订号，并在非触发型文档变更时取消旧补全，避免等长替换后显示过期 ghost text。
- 将 BEditor 内联补全的光标取消逻辑改为位置感知，并补齐输入法组合期间 documentChange 结束后的触发，避免同位置 cursor 噪声中断补全。
- 修复 Rich 内联补全 prompt 坐标与 Markdown 文本偏移不一致、Source 只读切换残留 ghost text，以及接受失败和空上下文触发后的状态机清理问题。
- 修复 Rich 模式撤销/重做快捷键被手写拦截和全局快捷键去重影响，导致长按无法连续回退的问题。
- 将 BEditor 内联补全状态机重命名为内部 `createInlineCompletionStateMachine`（不再导出），并把 pane 编排器从 `usePaneInlineCompletion` 重命名为导出的 `useInlineCompletion`，同步合并测试用例。
- 将阿里云服务商升级为独立 `alibaba` 请求格式，并支持把通用 reasoning 开关映射为 DashScope `enable_thinking` 参数。
