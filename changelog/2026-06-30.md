# 2026-06-30

## Added
- 添加 BPromptEditor 变量树可见节点计算与变量树展开/折叠交互回归测试。

## Changed
- 默认布局主内容区支持拖拽打开文件，欢迎页不再单独承载 DropZone。
- 默认布局主内容拖拽遮罩层级高于编辑器选区浮层，避免拖拽文件时 SelectionToolbar 露在遮罩上方。
- BChat 文件拖拽监听从输入框容器扩展到聊天容器，拖入聊天区域时仍高亮输入框。
- BPromptEditor 变量菜单支持树形变量节点展开/折叠，搜索嵌套变量时保留祖先路径上下文并继续允许父节点插入。
- BPromptEditor 变量树展开按钮改用 BButton，并移除变量 Chip 与选区样式中的未定义主题变量引用。
- BPromptEditor 变量树叶子节点仅在同层存在可折叠节点时展示折叠按钮占位。
- 修复 BPromptEditor 变量树叶子节点无折叠占位时内容被压缩到按钮列的问题。
- 修复 BPromptEditor 变量树叶子节点移除折叠占位后与父节点视觉并排的问题。
- 小组件执行方法编辑器的结果构造器提示拆分为命名接口，并补充各返回方法的 hover 说明与参数说明。
- BPromptEditor 默认保留变量 token 原文，不再把 `{{...}}` 自动渲染成变量 Chip，便于继续输入深层路径。
- 修复 BPromptEditor 变量菜单无匹配项时仍显示空白下拉框的问题。
- BPromptEditor 默认变量 token 改为主题色字体高亮，并保持文本可继续编辑。
- 小组件元素变量候选中的 `state.*` 路径改为从执行方法代码里的 `setState` 调用推导。
- 小组件设置面板的执行方法摘要改为可滚动的代码预览。
- 小组件设置面板的执行方法摘要预览改为复用 lowlight 与代码块高亮主题。
- BPromptEditor 变量菜单改为主文本展示变量路径，右侧仅在存在说明标签时展示 label。
- 小组件从执行方法推导 `state.*` 变量时仅复用类型结构，不再继承 input 字段说明。
- BPromptEditor 变量树子节点改为仅展示当前路径片段，减少父路径重复。
- 修复小组件 `state.*` 推导会误识别非 ctx `setState` 调用以及 input alias 作用域泄漏的问题。
- 修复小组件执行方法摘要预览未按真实代码换行展示的问题。
- 修复小组件执行方法摘要预览缺少代码高亮色值导致高亮不生效的问题。
- 小组件执行方法摘要预览支持按容器宽度自动换行。
- 小组件元素内容绑定对象或数组时改为展示格式化 JSON，避免出现 `[object Object]`。
- 小组件执行方法从 `metadata.skill.methods.execute` 简化为顶层 `execute`，降低配置协议层级。
- 小组件执行方法编辑器根据入参和出参 schema 生成 Monaco 类型提示。
- 修复小组件执行方法默认示例与出参 schema 类型提示不一致的问题，并在 schema 变化时重建方法编辑器类型声明。
- WebView Agent 操作不再仅因快照读取时间超过 60 秒而判定过期，仍保留 snapshotId、URL 与元素指纹校验。
- 修复 BChat 中止运行时后迟到的工具确认请求仍弹出确认卡片的问题。
- 修复小组件文本元素 resize 起手控制框与尺寸未对齐，以及预览被内容自适应高度反向覆盖导致拖拽闪烁的问题。
- 修复小组件文本元素 resize 过程中预览尺寸未经过文本内容测量，导致拖动时不重排、松手后才更新的问题。
- 修复小组件绑定内容文本元素 resize 测量未使用预览渲染上下文，导致起手尺寸与最终高度和实际渲染不一致的问题。
- 修复小组件文本元素移动起手时未恢复真实渲染尺寸，导致 Moveable 控制框高度小于文本内容的问题。
- 修复 Markdown 批注高亮下划线在跨行或行内代码片段中断开的问题。
- 修复 Markdown 批注卡片点击展示时未按完整批注范围定位，导致卡片位置偏离批注文本的问题。
- 修复 Markdown 批注覆盖行内代码后代码文本额外显示评论下划线的问题。
- 修复 Markdown 刷新后批注内部加粗、行内代码等语法未恢复为富文本的问题。
- 修复 Markdown 批注内行内代码背景遮挡共享批注下划线，导致连续线回退的问题。
- 修复 Markdown 批注恢复时普通 `[` 被错误并入后续批注范围的问题。

## Removed
- 移除小组件 PageSetter 的动态预览配置区，不再在设置面板中编辑 `metadata.previewContext`。
- 移除小组件旧根变量的类型、模板解析、预览上下文、方法提示与设计文档声明，仅保留 `input`、`state`、`output`。
