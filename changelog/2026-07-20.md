# 2026-07-20

## Changed

- `BText` 组件库整体更名为 `BSmart`：目录 `src/components/BText/` → `src/components/BSmart/`、测试目录 `test/components/BText/` → `test/components/BSmart/`，涉及 Editor / Input / Select / Method 四个组件及其类型导出（`BTextEditor*` → `BSmartEditor*`、`BTextSelect*` → `BSmartSelect*`、`BTextMethod*` → `BSmartMethod*`）。同步更新 `createNamespace` 命名空间（`text-*` → `smart-*`）与全部 CSS 类名（`b-text-*` → `b-smart-*`），以及 `vite.config.ts` 的 `COMPONENT_DIRS` 注册、`CONTEXT.md` 组件清单与目录树、所有外部引用（BChat / BWidget / views）。更名动机：原 `BText` 语义过弱，易被误解为纯文本展示组件，且与 `BWidget/elements/Text` 元素混淆；`BSmart` 更准确体现斜杠命令、变量芯片、@ 文件提及等富编辑能力。
- `BSmart/Editor.vue` 接入 `createNamespace('smart-editor')`，与 Input/Select/Method 三个同级组件保持一致：模板中硬编码的 `b-smart-editor-shell` / `b-smart-editor` / `b-smart-editor__container` / `b-smart-editor__codemirror` 改为 `:class="bem('shell')"` / `:class="name"` / `:class="bem('container')"` / `:class="bem('codemirror')"`；同时把非标准 BEM 类名 `b-smart-editor-shell`（单横线）规范为 `b-smart-editor__shell`（双下划线元素）。
- `WIDGET_DEFAULT_ELEMENT_STYLE` 新增 `lineHeight: 1` 默认值，新创建的 Widget 元素初始行高为 1，避免渲染时回退到 CSS `normal`（约 1.2）与设计预期不一致。
- `DesignSetter.vue` 行高 `BInputNumber` 增加 `:default-value="1"`，用户清空输入时兜底写入 1，避免 `lineHeight` 变为 `undefined`。
- BWidget 渲染与内容测量链路统一使用 options 对象传递渲染上下文与模式：新增 `renderOptions.ts` 的 `WidgetRenderEvaluationOptions`（`{ renderContext?, renderOptions?: { mode } }`），`getWidgetShapeRenderSize` / `measureContent` / `resolveWidgetTemplateFieldText` 等签名由位置参数收敛为 options 对象，避免继续增加位置参数；`useElementValue`、`WidgetNode`、`MoveableLayer`、`layout` 等调用点同步迁移。

## Fixed

- `BInputNumber` 修复 `defaultValue` 在初始渲染和外部置空时不作为显示兜底的设计缺陷：当 `v-model:value` 为 `undefined` 且设置了 `defaultValue` 时，输入框现在会显示 `defaultValue`，而非空白。此前 `defaultValue` 仅在用户触发 `update:value` 时参与输出归一化，名字与实际行为不符。
- `BInputNumber` 新增 `blur` 事件处理：用户清空输入时 `inputValue` 为空但 `modelValue` 已写入 `defaultValue`，失焦时把展示值同步回 `modelValue`，让 UI 与对外值一致；未配置 `defaultValue` 时保持空展示，不主动产生值。
- `ServiceConfig.vue` 恢复 `BButton` / `BModal` / `BModelIcon` / `BSelect` / `BSmartEditor` 的显式 import：`BText → BSmart` 重命名时误删了这些导入，导致 vitest 环境（未启用 `unplugin-vue-components`）下组件无法解析，`service-config.test.ts` 中 `.select-model` 选择器找不到 DOM。
- `session-id-runtime.test.ts` 的 `skillStoreMock` 补充 `syncDirtyFromDisk` mock 方法：`useRuntimeTools.syncAIResources` 调用了 `skillStore.syncDirtyFromDisk()`，但测试 mock 仅注册了 `syncFromDisk`，导致 23 个 BChat runtime 用例连锁失败。
- `node-click-selection.test.ts` 同步 `WIDGET_DEFAULT_ELEMENT_STYLE` 新增的 `lineHeight: 1` 默认值到断言期望对象。
- 修复 BWidget 文本元素在编辑态隐藏变量后仍按模板表达式撑大选区、导致手动缩放回退的问题：新增 `resolveWidgetDisplayValue`，按当前渲染模式解析字段展示值（design 模式剔除 `{{ }}` 变量仅留静态文本，runtime 模式按模板求值），文本测量改以当前模式可见内容为基准，不再因隐藏变量而撑大选区。
- 更新 `widget-canvas.component.test.ts` 的过时测试：原 `measures text node size from resolved render context content` 期望设计态也按模板解析变量测量，与「设计态隐藏变量测量」的新行为冲突，改为 `measures design-mode text node size from static fragments only`，验证设计态下混合文本只按静态片段测量、选框不被解析内容撑高。
