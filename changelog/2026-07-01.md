# 2026-07-01

## Changed
- Rich 编辑器加载或挂载 Mermaid 代码块时兼容内容行末尾粘连结束围栏的外部 Markdown，避免后续 5.2 等章节被误吞进代码块。
- Rich 编辑器与消息 Markdown 的 Mermaid 预览兼容结束围栏与后续标题粘连的外部文档写法，并在富文本代码块挂载时将误吞的后续标题恢复为独立标题节点。
- Skill 与 Widget 扫描在用户目录不存在时先跳过目录读取，避免编辑器刷新时触发 `fs:readWorkspaceDirectory` 主进程错误日志。
- 小组件JS 脚本改为 `Widget({ mounted, unmounted, methods })` 配置风格，并通过 `this.$input`、`this.$setState`、`this.$sendMessage` 提供上下文能力。
- 小组件JS 脚本编辑器类型提示移除旧的 `execute(ctx)` 和执行结果工厂，改为提示生命周期、methods 与上行消息参数结构。
- 小组件状态变量推导改为基于 `this.$setState(...)` 和 `this.$input...`，与新的JS 脚本协议保持一致。
- 小组件设置页新增 `constants` 常量目录，并将JS 脚本标题、默认代码、默认超时和 schema 默认字段名收敛到常量文件。
- 小组件JS 脚本的 `this.$state` 类型提示改为使用脚本中 `$setState` 推导出的状态 schema，并补充 TypeScript 诊断测试覆盖 `this` 上下文。
- 小组件JS 脚本编辑器支持根据当前草稿实时刷新 `this.$state` 类型提示，聊天 Widget part 协议改为使用所在消息 ID 表示独立运行态。
- 聊天消息内小组件支持 `created -> mounted` 最小运行闭环，`mounted` 中的 `this.$setState` 会写回宿主消息并持久化。
- BChat 消息级交互改为由底层组件适配统一 submit action，顶层聊天组件不再区分用户选择和小组件结果的事件数据。
- 小组件运行态消息写回复用 BChat 统一 submit action，不再向聊天顶层暴露 Widget 专属事件。
- 小组件 mounted 运行态改为受限解析 `this.$setState`，消息写回改为基于最新消息的 updater，并收窄 Widget memo key 成本。
- 小组件 mounted 状态初始化方法命名为 `initWidgetMountState`，消息气泡运行态入口命名为 `initWidgetRuntime`。
- 聊天小组件提交时会先执行受限 `unmounted` 状态写回，将原 Widget part 标记为 `finished` 后再发送 `widget_result`。
- 小组件 `unmounted` 支持受限解析 `this.$sendMessage(...)`，提交时可直接上行文本消息并跳过默认 `widget_result`。
- 小组件页面设置内的JS 脚本代码状态命名收敛为 `interactionScriptCode`。
- 小组件消息提交收尾逻辑下沉到 `BubblePartWidget`，`MessageBubble` 只负责传递消息定位与统一提交动作。
- 聊天消息结构化 part 新增必填稳定 `id` 字段，新建 part 时直接使用 `nanoid()` 生成，不再在读取历史消息时补齐。
- 小组件 `$sendMessage` 的数组文本 content 不携带 part id，运行态通过 `Omit<ChatMessageTextPart, 'id'>` 派生脚本文本片段类型避免重复声明，并由 `BubblePartWidget` 生成聊天消息 part id。
- Widget Skill 设计文档移除自然语言路由器、置信度候选和自动启动规划，改为只保留 `widget` / `open_widget` 工具打开流程。
- `widget` 与 `open_widget` 工具改为只从已启用且解析成功的小组件列表中命中，避免禁用小组件被模型打开。
- 小组件跨层协议类型收口到 `types/widget.d.ts`，并新增共享协议归一化工具统一处理 `open_widget` 展示载荷、提交结果和 `$sendMessage` 内容。
- 小组件JS 脚本补充受控 methods 与 `$http` 客户端，HTTP 请求通过通用 `request` 底层能力统一管理超时、主进程队列和代理，主进程 request 实现拆分到 `request/core`，队列只限制并发不限制等待长度，并支持字符串与特殊请求体直传。
- 小组件 `$http` 客户端创建逻辑收回到 `widgetRuntime`，移除独立 `widgetHttpClient` 文件和额外 method factory。
- 小组件JS 脚本移除脚本层 timeout 配置与运行态超时包裹，网络请求超时统一交给底层 `request` 能力处理。
- 修复小组件命名方法不存在时误进入等待状态的问题，并将托管 request 响应体大小限制改为流式读取时即时中止。
- 聊天小组件运行态写回移除 `hostPartId` 透传，统一使用当前 part id 定位消息片段。
- 聊天小组件运行态切换保持普通 part key，并在中断恢复时同步取消 `tool.widget` 内的未完成运行态。
- BWidget 视口尺寸 hook 支持通过静态模板 ref 名称绑定根节点，运行态组件不再因仅模板使用 `rootRef` 触发未读取诊断。
- BWidget 运行态控制器 prop 补充显式默认值，满足 Vue 可选 prop 默认值检查。
- 小组件 methods 触发 `$sendMessage` 结束时统一执行 `unmounted` 收尾，并将元素侧 method 调用改为触发型运行态动作。
- 小组件元素侧运行态接口改为 `runInteraction(interactionCode)`，系统只运行元素声明的交互表达式并以 `$sendMessage` 作为结束信号，不再把业务方法名作为平台协议。
- 小组件交互表达式调用用户辅助函数时支持实参求值与形参绑定，避免 `selectCoffee('latte')` 这类元素交互参数丢失。
- 小组件 `$http` 客户端类型提示中各方法的请求参数名由 `request` 改为 `options`，与通用 HTTP 调用习惯保持一致。
- 小组件JS 脚本编辑器的 `createWidgetMethodScriptTypes` 与 `createWidgetSchemaInterfaceDeclaration` 等 schema → TypeScript 接口生成工具抽离到 `src/views/widget/constants/methodScriptTypes.ts`，便于在编辑器与测试间复用。
