# 2026-07-01

## Changed
- 小组件交互脚本改为 `defineConfig({ mounted, unmounted, methods })` 配置风格，并通过 `this.$input`、`this.$setState`、`this.$sendMessage` 提供上下文能力。
- 小组件交互脚本编辑器类型提示移除旧的 `execute(ctx)` 和执行结果工厂，改为提示生命周期、methods 与上行消息参数结构。
- 小组件状态变量推导改为基于 `this.$setState(...)` 和 `this.$input...`，与新的交互脚本协议保持一致。
- 小组件设置页新增 `constants` 常量目录，并将交互脚本标题、默认代码、默认超时和 schema 默认字段名收敛到常量文件。
- 小组件交互脚本的 `this.$state` 类型提示改为使用脚本中 `$setState` 推导出的状态 schema，并补充 TypeScript 诊断测试覆盖 `this` 上下文。
- 小组件交互脚本编辑器支持根据当前草稿实时刷新 `this.$state` 类型提示，聊天 Widget part 协议改为使用所在消息 ID 表示独立运行态。
- 聊天消息内小组件支持 `created -> mounted` 最小运行闭环，`mounted` 中的 `this.$setState` 会写回宿主消息并持久化。
- BChat 消息级交互改为由底层组件适配统一 submit action，顶层聊天组件不再区分用户选择和小组件结果的事件数据。
- 小组件运行态消息写回复用 BChat 统一 submit action，不再向聊天顶层暴露 Widget 专属事件。
- 小组件 mounted 运行态改为受限解析 `this.$setState`，消息写回改为基于最新消息的 updater，并收窄 Widget memo key 成本。
