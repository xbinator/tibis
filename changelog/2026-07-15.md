# 2026-07-15

## Changed

- 移除小组件设置页的重复 Store 初始化，统一由默认布局负责资源扫描与目录监听。
- 将 Skill 与 Widget 资源初始化 Hook 迁移至默认布局目录，明确其应用布局级职责。
- 抽取公共 `src/hooks/useFileStoreInit.ts`，统一 Skill / Widget Store 初始化的「准备屏障 → 订阅事件 → 监听目录 → 扫描 → 清理」流程；`useSkillInit` 与 `useWidgetInit` 改为纯配置委托。
- 将 `src/shared/workspace/pathUtils.ts` 迁移到 `src/utils/file/`：`posix.ts` 补齐 `join` / `basename` / `parseParts` / `slashify`，新增 `path.ts`（路径片段安全校验）与 `workspace.ts`（工作区根解析与包含判断）。所有集合统一为 `export const xxx = { ... }` 形式（仿 `posix`），函数命名进一步收敛：`resolvePathAgainstWorkspace` → `workspace.resolveWithin`，`isPathInsideWorkspace` → `workspace.contains`（参数顺序由 `(target, root)` 调整为 `(root, target)`），`isSafeFilePathSegment` → `path.isValidSegment`，`normalizeSafeRelativeFilePath` → `path.validatePath`，`normalizeSeparators` → `posix.slashify`。
- 资源监听 hook 命名收敛：通用 `useFileStoreInit` / `FileStoreInitConfig` / `createTargetFileMatcher` 重命名为 `useWatchResource` / `WatchResourceConfig` / `createResourceMatcher`，同步重排为 `src/hooks/useWatchResource.ts`；布局级 `useSkillInit` / `useWidgetInit` 同步重命名为 `useWatchSkillResource` / `useWatchWidgetResource`，新文件分别位于 `src/layouts/default/hooks/useWatchSkillResource.ts` 与 `useWatchWidgetResource.ts`。语义强调「监听用户目录中的资源并同步到 Store」，弱化「Store 内部初始化」这一被误读的语义。
