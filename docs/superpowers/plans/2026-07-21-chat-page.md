# Independent Chat Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `/chat` and `/chat/:sessionId` pages that reuse `BChat`, support multiple chat tabs, keep ChatSider session ownership exclusive, and surface background runtime state in HeaderTabs.

**Architecture:** Route helpers provide one source of truth for chat paths and tab identities. A renderer-only Pinia store tracks chat-tab runtime ownership, status, and abort controllers without persisting them. `src/views/chat/index.vue` is a thin `BChat` host that promotes `/chat` to `/chat/:sessionId`, while ChatSider and HeaderTabs use the same registry for navigation, deletion, and close protection.

**Tech Stack:** Vue 3.5, Vue Router 5, Pinia 3, TypeScript 5.9 strict mode, Vitest 4, Less.

```ts
/**
 * @file chatRouteTab.ts
 * @description 统一解析聊天页路径、标签身份与会话归属。
 */
import type { Tab } from '@/stores/workspace/tabs';

/** 空白聊天页固定标签 ID。 */
export const CHAT_DRAFT_TAB_ID = 'chat:new';
/** 聊天标签 ID 前缀。 */
export const CHAT_TAB_PREFIX = 'chat:';

/**
 * 构建聊天页路径。
 * @param sessionId - 持久化会话 ID，空值表示草稿页
 * @returns 聊天页绝对路径
 */
export function createChatPath(sessionId?: string | null): string {
  return sessionId ? `/chat/${encodeURIComponent(sessionId)}` : '/chat';
}

/**
 * 构建聊天标签 ID。
 * @param sessionId - 持久化会话 ID，空值表示草稿页
 * @returns 聊天标签 ID
 */
export function createChatTabId(sessionId?: string | null): string {
  return sessionId ? `${CHAT_TAB_PREFIX}${sessionId}` : CHAT_DRAFT_TAB_ID;
}

/**
 * 从聊天路径解析会话 ID。
 * @param path - 标签或路由路径
 * @returns 会话 ID；草稿或非聊天路径返回 undefined
 */
export function resolveChatSessionId(path: string): string | undefined {
  const matched = /^\/chat\/([^/?#]+)(?:[?#].*)?$/u.exec(path);
  if (!matched?.[1]) return undefined;
  try {
    return decodeURIComponent(matched[1]);
  } catch {
    return matched[1];
  }
}

/**
 * 查找指定聊天会话对应的标签。
 * @param tabs - 当前标签列表
 * @param sessionId - 会话 ID；空值查找草稿标签
 * @returns 命中的聊天标签
 */
export function findChatTab(tabs: Array<Pick<Tab, 'id' | 'path' | 'title'>>, sessionId?: string | null): Pick<Tab, 'id' | 'path' | 'title'> | undefined {
  const targetId = createChatTabId(sessionId);
  return tabs.find((tab) => tab.id === targetId);
}
```

Create two route records. The persisted route uses resolver functions that normalize `route.params.sessionId`, both routes set `props`, and both use `lucide:message-circle`.
