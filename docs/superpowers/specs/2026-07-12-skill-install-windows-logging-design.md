# Skill 安装 Windows 容错与日志增强设计

## 背景

Skill 导入包在 macOS 可以完成安装，但 Windows 可能在临时目录替换为正式目录时失败。当前安装流程会把底层目录重命名异常替换成通用提示，导致 `EPERM`、`EACCES`、`EBUSY` 等 Windows 错误码无法通过“设置 → 运行日志”定位。

## 目标

- 统一 Skill 安装路径拼接方式，兼容 Windows 与 POSIX 路径。
- 对 Windows 常见的瞬时目录占用错误执行有限次数重试。
- 在安装关键阶段写入持久化日志，并保留最终失败的原始错误信息。
- 让运行日志页面可以主动刷新，便于安装失败后立即查看最新记录。
- 将安装操作按钮文案和测试统一为“确定”。

## 非目标

- 不新增专用 Skill 安装 IPC。
- 不改变 Widget 安装流程。
- 不修改现有日志文件格式、筛选规则或脱敏规则。
- 不对所有文件系统重命名操作增加全局重试。

## 方案

### 安装路径

`src/views/settings/tools/skill/components/SkillCreator.vue` 复用 `src/ai/skill/parser.ts` 导出的 `joinPath`，移除组件内的简单字符串拼接函数。所有安装目录、入口文件和资源文件路径继续统一使用 `/` 表示，Electron 主进程在 Windows 上交给 Node 文件系统解析。

### 重命名重试

在 Skill 安装组件附近提供只服务于安装事务的目录重命名辅助逻辑：

- 首次调用立即执行。
- 仅当错误消息包含 `EPERM`、`EACCES` 或 `EBUSY` 时重试。
- 使用固定的短间隔，最多执行有限次数，避免安装按钮长时间无响应。
- 非瞬时错误立即抛出。
- 最终失败时抛出原始错误，不再用通用错误覆盖。

该逻辑同时用于“正式目录 → 备份目录”“临时目录 → 正式目录”和回滚操作，确保覆盖安装的各个目录切换阶段行为一致。

### 阶段日志

安装组件通过 `src/shared/logger/index.ts` 的持久化 Logger 写入以下记录：

- INFO：开始安装、临时内容写入完成、旧版本备份完成、目录替换完成、重新扫描完成、安装成功。
- WARN：瞬时目录占用触发重试、备份清理失败、重新扫描失败、回滚执行。
- ERROR：安装最终失败，包含当前阶段、Skill 名称和序列化后的原始错误。

日志中的路径继续经过现有主进程脱敏逻辑，不新增未经处理的敏感信息。日志调用自身失败不能改变安装结果，因此日志写入使用安全包装，避免日志系统异常中断安装事务。

### 用户提示

- 安装成功仍显示成功消息。
- 安装完成但列表扫描失败时仍显示警告，不误报为安装失败。
- 安装失败提示包含原始错误摘要，使用户无需打开开发者工具即可获得基本线索。
- 安装按钮统一显示“确定”。

### 运行日志页面

`src/views/settings/logger/index.vue` 在页面操作区增加刷新按钮。刷新时重新加载可用日期和当前筛选条件下的第一页日志，并复用现有 loading 状态防止并发请求。

## 数据流

1. 用户上传并解析 Skill 包。
2. 用户点击“确定”，组件记录安装开始。
3. 组件在用户 Skill 根目录创建临时目录并写入文件。
4. 目标目录存在时，将其重命名为备份目录；瞬时占用错误进入有限重试。
5. 将临时目录重命名为目标目录；失败时尝试回滚备份。
6. 清理备份并重新扫描 Skill 列表。
7. 各阶段写入持久化日志，Logger 页面可主动刷新显示。

## 错误处理

- 获取用户目录失败：记录 ERROR，并显示原始错误摘要。
- 临时文件写入失败：清理临时目录，记录失败阶段。
- 备份失败：保留正式目录，不进入替换阶段。
- 替换失败：尝试恢复备份；分别记录替换错误与回滚结果。
- 备份清理失败：安装仍成功，记录 WARN，后续扫描器继续负责孤儿目录清理。
- 列表扫描失败：磁盘安装仍视为成功，记录 WARN 并提示稍后重试。

## 测试

- 组件测试使用 `C:\\Users\\test` 验证统一路径输出。
- 验证瞬时重命名错误会重试并最终安装成功。
- 验证非瞬时错误不会重试。
- 验证最终失败提示和 ERROR 日志保留原始错误码。
- 验证安装成功与扫描失败分别产生正确日志。
- 验证按钮文案为“确定”。
- Logger 页面测试验证刷新操作会重新请求日志文件和日志条目。

## 验收标准

- Windows 路径不会产生重复分隔符或依赖组件内手写拼接。
- 瞬时目录占用可在有限重试后恢复。
- 最终失败时界面与运行日志均能看到原始错误摘要。
- Logger 页面可主动刷新最新日志。
- 相关单元测试、TypeScript 类型检查、ESLint、Stylelint 和构建通过。
