# AskUserChoiceCard 步骤条改造设计

## 背景

将 `AskUserChoiceCard.vue` 从单卡片形式改造为两步步骤条，提升用户体验。

## 目标

- 第一步：展示 LLM 返回的问题选项
- 第二步：固定问题"是否有更多的补充信息需要提供？（可选）" + 输入框

## 设计细节

### 交互流程

```
步骤1: 选择答案          步骤2: 补充信息
┌─────────────────┐     ┌─────────────────┐
│ 问题标题         │     │ 是否有更多的补充 │
│ ○ 选项A         │ ──► │ 信息需要提供？   │
│ ○ 选项B         │     │ [输入框]        │
│ [下一步]        │     │ [上一步] [提交]  │
└─────────────────┘     └─────────────────┘
```

### 组件改造

| 项目 | 当前 | 改造后 |
|------|------|--------|
| 步骤条 | 无 | 使用 `a-steps` 组件 |
| 第一步 | 选项 + "其他"输入框 | 仅选项，底部按钮改为"下一步" |
| 第二步 | 无 | 固定问题 + 输入框，底部"上一步" + "提交" |
| "其他"输入框 | 第一步有 | 移到第二步 |

### 数据流

- 第一步选择存入 `selectedValues`
- 第二步输入存入 `otherText`
- 点击"提交"时一次性 emit `submit-choice`

### 关键逻辑

1. **第一步验证**：必须选择至少一项才能点"下一步"
2. **第二步可跳过**：输入框可空，直接点"提交"
3. **回退保留状态**：返回第一步时，之前的选择保留
4. **数据结构不变**：`AIUserChoiceAnswerData` 保持原有结构

## 实现要点

1. 使用 `ant-design-vue` 的 `a-steps` 组件
2. 新增 `currentStep` 状态控制当前步骤
3. 第一步移除"其他"输入框，按钮改为"下一步"
4. 新增第二步 UI，包含固定问题和输入框
5. 按钮文字根据步骤动态变化
