# BChat 斜杠技能引用设计

## 背景

`src/components/BChat/index.vue` 已通过 `useSlashCommands` 向 `BTextEditor` 提供 `/model`、`/usage`、`/new` 和 `/compact` 等静态命令。当前斜杠菜单仅在当前行以 `/` 开头时生效，技能则只通过 `useSkillStore` 暴露的 `skill` 工具由模型按需选择，用户无法从输入框明确指定本轮要使用的技能。

本功能需要把已启用技能直接加入同一个斜杠菜单。用户选择技能后，输入框以轻量的 `SkillReference` 形式显示 `$技能名`；提交时 Runtime 必须确定性加载所选技能，而不是依赖模型看到普通文本后自行猜测是否调用 `skill` 工具。

## 目标

- 用户在输入内容开头或空白字符后输入 `/` 时，同时看到“命令”和“技能”两组候选项。
- 斜杠查询可以出现在输入内容任意位置，选择后只替换当前查询范围，不破坏其他正文。
- 技能候选项来自 `useSkillStore()` 中已启用且无解析错误的技能，并随 Store 变化实时更新。
- 选择技能后显示 `$技能名`，允许同一消息选择任意多个技能。
- 技能引用使用结构化数据持久化，支持特殊字符、整体删除、编辑恢复、重新生成和会话恢复。
- Runtime 确定性加载本轮所选技能的最新内容，并在同一轮工具续跑或用户交互续跑期间持续生效。
- Skill 内容保持用户级指令权限，不提升为 system 指令。
- `SkillReference` 在输入框和消息气泡中使用一致的轻量主题样式。

## 非目标

- 不增加 `/skill` 二级选择流程。
- 不限制单条消息可选择的技能数量。
- 不改变设置页的技能扫描、启用、禁用、创建或删除行为。
- 不改变模型自主调用现有 `skill` 工具的能力。
- 不让一次显式技能选择自动变成整个会话的持久技能配置。

## 术语与命名

- 视觉与业务名称统一使用 `SkillReference`，不使用 `Chip`、`Tag`、`Badge` 或 `SkillMention`。
- 消息结构类型使用 `ChatMessageSkillReferencePart`。
- 技能引用工具模块使用 `skillReference`。
- 样式根类使用 `b-skill-reference`。
- 斜杠菜单技能项稳定 ID 使用 `skill:${name}`，与内置动作命令 ID 隔离。

## 交互设计

### 斜杠触发范围

斜杠上下文只在以下条件同时满足时成立：

1. 编辑器选择区为空。
2. 光标前最近的候选 `/` 位于输入开头、当前行开头或空白字符之后。
3. 候选 `/` 到光标之间不包含空白字符或另一个 `/`。
4. 当前查询至少匹配一个命令或技能候选项；空查询匹配全部候选项。

因此 `帮我用 /weather 查询上海` 会打开菜单并把 `/weather` 作为活动范围；`https://example.com`、`src/foo.ts` 和 `2026/07/14` 不会触发。选择条目时只删除活动范围，活动范围之前和光标之后的文本保持不变。

### 候选项与过滤

斜杠菜单按以下顺序分组：

1. 命令：现有静态动作命令。
2. 技能：当前已启用且无解析错误的技能。

空查询展示两组全部条目。非空查询不区分大小写，匹配触发文本前缀，也匹配标题和描述中的文本。键盘上下移动、Enter 选择、Escape 关闭、鼠标悬停和活动项滚动继续使用同一个扁平索引；分组标题不参与索引计算。

内置命令和技能允许显示相同触发文本，因为稳定 ID 和分组不同。选择内置命令时保持现有动作；选择技能时插入结构化技能引用且不自动提交消息。

### SkillReference 视觉

`SkillReference` 是正文内联引用，不使用胶囊视觉：

- `display: inline-flex`，与正文基线对齐。
- 左侧使用 `lucide:wrench` 图标，图标尺寸为 `14px`。
- 图标与 `$技能名` 均使用 `var(--color-primary)`。
- 技能名称使用 `font-weight: 600`。
- 图标与文本之间保留紧凑间距。
- 不设置边框、背景色、圆角、阴影或额外内边距。
- Hover 只把颜色切换为 `var(--color-primary-hover)`，不出现背景或边框。
- 引用在编辑器中作为原子范围处理，一次退格或删除操作移除完整引用。
- 输入编辑器和已发送用户消息使用同一视觉规范。

## 数据模型

### 编辑器 Token

编辑器文档使用可逆 Token 保存稳定身份：

```text
{{$skill:<percent-encoded-name>}}
```

例如 `weather` 对应 `{{$skill:weather}}`。Token 中的技能名使用百分号编码，避免中文、空格和特殊字符破坏匹配边界。编辑器通过现有原子装饰基础设施把 Token 渲染为 `SkillReference`，用户看到的是 `🔧 $weather`，而不是内部 Token。

### 消息 Part

提交解析器把 Skill Token 转换为有序的 `ChatMessageSkillReferencePart`：

```typescript
interface ChatMessageSkillReferencePart extends ChatMessagePartBase {
  /** 片段类型 */
  type: 'skill_reference'
  /** Skill frontmatter 名称 */
  name: string
  /** 用户输入中的原始 Token 与位置 */
  sourceText: {
    /** Token 起始 offset */
    start: number
    /** Token 结束 offset */
    end: number
    /** 原始 Token 文本 */
    value: string
  }
}
```

消息 `content` 保存用户可读投影，例如 `$weather 查询上海天气`；完整 Skill 内容不写入消息数据库。消息 `parts` 保存技能引用与普通文本、文件引用的原始顺序。模型历史投影在非活动轮次中只把 Skill part 表示为 `$技能名`，不会再次注入 Skill 内容。

编辑消息或恢复草稿时，根据 `parts` 重建编辑器 Token；如果加载的是没有结构化 Skill part 的旧消息，则继续使用原 `content`，不尝试猜测普通 `$文本` 是否为技能。

## Runtime 数据流

### 首次发送

1. `BTextEditor` 选择技能后插入 Skill Token。
2. 输入解析器把 Token 转换为 `ChatMessageSkillReferencePart`，同时生成用户可读 `content`。
3. Runtime 准备阶段先同步 AI 资源，再按 part 中的技能名调用 Skill Store 的最新内容解析能力。
4. 任一技能已禁用、被删除或解析失败时终止准备，保留当前草稿并显示明确错误。
5. Renderer 把去重后的技能内容快照作为当前 Runtime 请求数据发送给主进程；消息持久化只保存引用 part。
6. 主进程在构建当前用户消息的模型投影时，把技能内容包装为明确的“用户显式选择技能上下文”，再拼接用户请求正文。
7. 技能内容作为当前 user 上下文参与模型调用，不拼接到 system prompt，也不伪造 assistant 工具调用或工具结果。

同一消息可以包含任意多个不同技能。重复选择同名技能时保留用户可见引用，但 Runtime 按首次出现顺序去重，只加载一次该技能内容。

### 轮次持续时间

Runtime 在活动状态中持有所选技能内容和目标用户消息 ID。模型工具续跑、确认续跑、用户选择续跑、上下文自动压缩后的继续执行，以及上下文溢出重放都继续向同一个活动用户轮次注入这些技能。

本轮完成后，技能内容不成为会话级配置。下一条普通用户消息只看到历史中的 `$技能名` 文本，不重新注入旧 Skill 内容。

### 重新生成与恢复

重新生成 assistant 消息时，从对应的最后一条用户消息读取 `ChatMessageSkillReferencePart`，同步磁盘并重新解析当前最新 Skill 版本，再创建新的 Runtime 请求。等待用户交互的 Runtime 在页面刷新后续跑时同样从持久化引用恢复技能选择。

如果原消息引用的技能已不可用，重新生成或恢复续跑会停止并提示具体技能名称，不会静默忽略该技能，也不会退化为普通 `$文本` 请求。

## 信任边界

Skill 可能来自用户安装的第三方目录。显式选择表示用户授权本轮读取其指令，但不应提升其指令优先级。因此完整 Skill 内容只能进入当前 user 模型上下文，不能写入 `system` 字段。

Skill 内容使用带名称、内容哈希和来源路径的定界结构包装，用户正文使用独立定界区域，降低内容与请求边界混淆的风险。名称和路径在构建定界文本时必须转义。Runtime 仍遵守已有工具权限、确认和沙箱规则；Skill 文本本身不能绕过工具执行权限。

## 上下文容量

产品层不设置技能数量上限。Runtime 上下文用量估算必须包含当前注入的全部去重 Skill 内容，保证工具栏用量和发送前压缩判断反映真实请求成本。

如果所选技能内容与不可压缩的基础上下文本身已经超过当前模型可用窗口，Runtime 返回明确的上下文超限错误，不截断、丢弃或随机选择部分技能。单个 Skill 继续遵守现有解析器的内容长度规则。

## 错误处理

- Skill Store 尚未完成初始化时，菜单仍显示内置命令；初始化完成后响应式补充技能。
- 技能在展示后、提交前发生变化时，以提交准备阶段的磁盘最新状态为准。
- 技能解析失败、禁用或删除时，发送和重新生成均中止并指出技能名称。
- Runtime 准备失败时不清空输入内容、图片或 Skill Token。
- 技能候选项为空时不展示“技能”空分组，不影响命令组。
- 查询无匹配项时关闭斜杠菜单，正文保持不变。
- Skill 内容导致上下文超限时使用专门错误文案，避免用户误认为模型或网络故障。

## 兼容性与迁移

- 现有斜杠动作命令 ID 和处理回调保持兼容。
- 现有普通文本、文件引用、图片、Widget 结果和历史消息不需要迁移。
- 旧消息中的 `$name` 始终视为普通文本，只有结构化 `skill_reference` part 具有确定性加载语义。
- 现有模型自主调用 `skill` 工具的流程继续可用，并继续使用内容哈希失效策略。
- 不支持工具调用的模型仍可使用显式选择技能，因为 Skill 内容由 Runtime 作为用户上下文直接提供。

## 测试策略

### BText 斜杠上下文

- 输入开头和空白字符后输入 `/` 会打开菜单。
- URL、文件路径和日期中的 `/` 不会打开菜单。
- 多行输入中的行首 `/` 可以触发。
- 选择条目只替换当前活动范围，前后正文保持不变。
- 标题与描述过滤、空查询、无结果关闭和键盘索引在分组后保持正确。

### 技能候选与展示

- 仅把已启用且无解析错误的技能转换为菜单项。
- Store 增删、启用和禁用后菜单响应式更新。
- 命令与技能重名时保持两个稳定条目。
- `SkillReference` 在编辑器和消息气泡中使用图标、主题色和加粗文本，且不存在边框与背景样式。
- Skill Token 支持中文、空格和特殊字符，并可整体删除和恢复。

### 消息与 Runtime

- 输入解析生成正确顺序的文本、文件和 `skill_reference` parts。
- 消息可读 `content` 使用 `$技能名`，数据库不持久化完整 Skill 内容。
- 多技能按首次出现顺序加载，同名技能只注入一次。
- 当前 Runtime 的首轮、工具续轮、用户交互续跑和溢出重放持续注入技能。
- 下一条普通消息不重新注入旧技能。
- 重新生成读取技能最新版本。
- 技能失效时保留草稿并阻止发送或续跑。
- 上下文用量包含 Skill 内容，无法容纳全部技能时返回明确错误。
