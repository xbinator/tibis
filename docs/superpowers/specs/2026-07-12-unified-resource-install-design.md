# Skill 与 Widget 统一安装事务设计

## 背景

Skill 与 Widget 都需要将入口文件和附带资源写入用户目录，但当前实现分散在两个页面：Skill 自行处理临时目录、备份、替换与回滚，Widget 直接创建目录并逐个写入。重复的文件事务增加维护成本，也让两种资源的错误提示、日志和失败恢复行为不一致。

本次同时同步用户调整后的运行日志刷新按钮测试，并统一日志筛选控件的视觉高度与字号。

## 目标

- 抽取与业务类型无关的目录安装事务，统一 Skill 与 Widget 的写入、冲突、替换、回滚、Windows 重试和清理。
- 通过配置表达 Skill 允许覆盖、Widget 禁止覆盖，不把差异散落在业务组件中。
- 两种资源保留各自表单和预览流程，只减少底层文件安装的心智负担。
- 两种安装流程使用一致的阶段事件、持久化日志和失败提示。
- Logger 刷新测试只约束稳定行为，不依赖图标、按钮类型或排列。
- Logger 筛选控件统一为 `28px` 高、`12px` 字体。

## 非目标

- 不合并 Skill 与 Widget 的创建弹窗。
- 不把覆盖策略做成用户可见的设置项。
- 不修改现有日志文件格式。
- 不改变 Skill 包与 Widget 包的解析格式。

## 统一目录安装器

新增通用目录安装模块，输入包含：

- 目标目录。
- 文本入口文件及二进制资源文件。
- 冲突策略 `replace` 或 `reject`。
- 文件系统 API。
- 安装阶段事件回调。
- 临时目录和备份目录名称生成器。

安装器只负责文件系统事务，不依赖 Vue、Pinia、消息组件或具体资源类型。

### 文件模型

安装文件统一为带相对路径的判别联合类型：

- `{ kind: 'text', relativePath, content }`
- `{ kind: 'binary', relativePath, content }`

相对路径由 `src/shared/workspace/pathUtils.ts` 统一校验。ZIP 解析器、Skill、Widget 与安装器共用相同的跨平台路径片段规则和路径拼接逻辑，保留 Windows UNC 前缀，避免重复实现发生语义漂移。

### 冲突策略

- `replace`：目标存在时先移动至备份目录，激活临时目录失败时恢复备份。Skill 使用该策略。
- `reject`：目标存在时立即抛出明确冲突错误，不写入、不移动、不清理目标目录。Widget 使用该策略。

策略是调用方的静态配置，不增加用户操作步骤。

### 事务流程

1. 检查目标目录状态。
2. 根据冲突策略拒绝或准备覆盖。
3. 在目标目录同级创建临时目录。
4. 创建父目录并写入所有文本、二进制文件。
5. `replace` 策略下将现有目标移动到备份目录。
6. 将临时目录激活为目标目录。
7. 激活失败时恢复备份。
8. 成功后清理备份；失败时清理临时目录。

安装开始前通过 Electron 主进程按目标路径获取跨窗口锁，并在目标同级写入事务记录。应用异常退出后，Skill/Widget 扫描器会在同一把锁内恢复备份或清理临时目录；清理失败时保留事务记录供下次重试。

目录重命名继续对 `EPERM`、`EACCES`、`EBUSY` 执行有限重试，最终失败保留原始错误。

## 业务调用方

### Skill

`SkillCreator.vue` 将 `SKILL.md` 和资源数组转换为统一文件模型，以 `replace` 策略调用安装器。安装完成后执行 Skill Store 扫描；扫描失败只记录警告，不误报磁盘安装失败。

### Widget

`widget/index.vue` 将 `widget.json` 和导入资源转换为统一文件模型，以 `reject` 策略调用安装器。安装完成后执行 Widget Store 扫描、更新 Store 并打开编辑器。

- 扫描失败：Widget 已安装，继续更新 Store 并显示降级警告。
- 打开编辑器失败：Widget 保留在磁盘和 Store，显示降级警告。
- 文件事务失败：显示原始错误并记录失败阶段。

## 日志

通用安装器通过事件回调报告检查、写入、备份、重试、激活、回滚和清理阶段。Skill 与 Widget 调用方分别使用 `[skill-install]` 和 `[widget-install]` 前缀写入现有持久化 Logger。

事务恢复失败同样通过非阻断回调写入持久化 Logger，日志故障不会改变安装或扫描结果。

安装器自身不直接依赖 Logger，使事务可独立测试并支持其他资源类型复用。

## Logger 刷新测试

保留用户当前按钮结构：筛选栏在前、刷新按钮在后，按钮不强制图标或类型。

测试验证：

- 页面存在文案为“刷新”的按钮。
- 点击后重新读取日志文件列表和当前筛选条件的第一页。
- 加载期间按钮禁用，避免重复刷新。

测试不检查图标、`type`、DOM 相邻顺序或实现类名。

## LogFilterBar 样式

`LogFilterBar.vue` 通过 scoped Less 对 BSelect、AInput 和 ADatePicker 的根容器及内部输入文字做局部覆盖：

- 控件视觉高度统一为 `28px`。
- 输入文字、占位文字和选择内容统一为 `12px`。
- 保留现有宽度、间距、颜色和交互状态。
- 不使用行内样式，不改变全局组件主题。

`ui-ux-pro-max` 对普通正文建议不小于 `12px`；本次使用 `12px` 作为紧凑设置工具栏的下限，并保留组件原有焦点、键盘和清除交互。

## 测试

- 通用安装器：写入文本/二进制文件、`replace`、`reject`、Windows 重试、回滚、临时目录清理、清理失败保留事务和崩溃恢复。
- 共享路径：POSIX、Windows 盘符、UNC 路径与危险相对路径。
- 主进程文件 API：仅把 `ENOENT`/`ENOTDIR` 识别为路径不存在，权限错误保持原始异常。
- Skill：文件模型映射、覆盖策略、阶段日志、扫描失败降级。
- Widget：文件模型映射、拒绝冲突、阶段日志、扫描和编辑器失败降级。
- Logger：用户修改后的刷新按钮行为与 loading 防重复。
- LogFilterBar：使用样式检查覆盖高度和字号，业务事件测试保持不变。

## 验收标准

- Skill 与 Widget 不再各自实现目录写入事务。
- 覆盖差异仅由 `conflictStrategy` 配置表达。
- Windows 重命名容错、失败回滚与日志行为一致。
- 用户调整后的刷新按钮测试通过且不脆弱。
- 三个筛选控件高度均为 `28px`，文字均为 `12px`。
- 全量测试、TypeScript、ESLint、Stylelint 和构建通过。
